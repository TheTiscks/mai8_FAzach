## Представление целых чисел
Современные процессоры и компилятор C используют почти исключительно дополнительный код (two’s complement) для signed типов.
Однако на зачёте часто требуют знать все три исторических формата:

Прямой код — старший бит = знак (0 положительное, 1 отрицательное), остальные биты — модуль числа
Пример на 8 битах: +5 → 00000101, -5 → 10000101

Недостатки — два нуля (+0 и -0), сложение требует отдельной обработки знаков
Обратный код (one’s complement) — положительные как в прямом коде, отрицательные — инверсия всех бит модуля
Пример на 8 битах: +5 → 00000101, -5 → 11111010

Недостатки — всё ещё два нуля, при сложении часто нужна end-around carry коррекция
Дополнительный код (two’s complement) — положительные как в прямом коде, отрицательные = инверсия всех бит модуля + 1
Пример на 8 битах: +5 → 00000101, -5 → 11111011

Преимущества — один ноль, сложение и вычитание одинаковы для всех чисел, переполнение легко обнаруживается
Именно этот формат используется во всех современных CPU для signed char, short, int, long, long long
unsigned типы — всегда прямой двоичный код, арифметика по модулю 2ⁿ


### Порядок байтов (endianness)

Little-endian — младший байт по младшему адресу (x86, x64)
Big-endian — старший байт по младшему адресу (сетевые протоколы, большинство RISC)


#### Представление вещественных чисел
Стандарт IEEE-754 (обязателен для всех современных компиляторов C)
Общая формула: (-1)^S × M × 2^E

float (32 бита)

1 бит знак — 8 бит экспонента (смещение 127) — 23 бита дробной части мантиссы
нормализованная мантисса = 1.xxx… (целая 1 не хранится, подразумевается)
double (64 бита)

1 бит знак — 11 бит экспонента (смещение 1023) — 52 бита дробной части мантиссы
нормализованная мантисса = 1.xxx…

Реальная экспонента = хранимая экспонента минус смещение

Мантисса всегда нормализована в диапазоне [1, 2), поэтому старший бит = 1 и не хранится
Специальные значения IEEE-754

+0 и -0 — отличаются только битом знака

+∞ и -∞ — экспонента все 1, мантисса все 0

NaN — экспонента все 1, мантисса ≠ 0 (quiet NaN и signalling NaN)
Денормализованные числа — экспонента все 0, мантисса ≠ 0 (позволяют представить числа ближе к нулю)


### Ключевые последствия для алгоритмов на C

0.1, 0.2, 0.3 и многие десятичные дроби не имеют точного представления → ошибки округления
Сравнивать float/double на == нельзя, только |a-b| < EPS
Сложение неассоциативно: (a + b) + c ≠ a + (b + c) при больших различиях в величине
float ≈ 7 десятичных знаков точности, double ≈ 15–16 знаков
