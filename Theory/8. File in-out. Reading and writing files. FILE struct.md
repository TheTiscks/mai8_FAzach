## Файловый ввод/вывод в языке C. Методы чтения и записи файлов. Циклы чтения файлов с обработкой символа конца файла и без неё. Типизированные и не типизированные файлы. Структура FILE. Форматируемый ввод/вывод. Методы выделения лексем из входного строкового потока. Принцип границы и функция strtok.

Файловый ввод/вывод позволяет программам читать данные из файлов и записывать данные в файлы. 

В C для работы с файлами используется стандартная библиотека stdio.h.

#### Методы чтения и записи файлов

Основным понятием при работе с файлами является указатель на файл типа FILE. Этот указатель используется для всех операций чтения и записи.

Функция fopen открывает файл и возвращает указатель на него

### Структура FILE и базовые операции

FILE — непрозрачная структура из <stdio.h>, содержит всю информацию о потоке (буфер, позиция, флаги, дескриптор и т.д.)
Открытие файла:
``fopen(const char *filename, const char *mode)``

Основные режимы:

"r" — чтение, "w" — запись (создаёт/перезаписывает), "a" — дозапись,

"+" — добавляет чтение/запись, "b" — бинарный режим (обязательно в Windows)

Закрытие: ``fclose(FILE *fp)`` — обязательно, иначе потеря данных из буфера

Проверка ошибок: ``if ((fp = fopen("file.txt", "r")) == NULL) → выход/обработка``

<hr>

* Типизированные файлы — это текстовые файлы.
Данные сохраняются в виде символов (например, число 12345 → символы '1','2','3','4','5').
Используются функции с форматной строкой: fprintf, fscanf и их аналоги.

* Нетипизированные (бинарные) файлы — данные сохраняются точно в том виде, в каком лежат в памяти.
Можно записать и прочитать структуру, массив или целый блок памяти одной операцией fread/fwrite.

#### Типизированный (форматированный) ввод/вывод
Работает с текстом, автоматически преобразует типы.

fprintf(fp, format, …) — запись

fscanf(fp, format, …) — чтение

printf / scanf — просто используют stdout / stdin

sprintf / sscanf — работают со строками как с файлами

#### Нетипизированный (бинарный) ввод/вывод

Работает с сырыми байтами, без преобразования.
````
fread(void *buf, size_t size, size_t count, FILE *fp)
fwrite(const void *buf, size_t size, size_t count, FILE *fp)
````

Пример записи структуры за одну операцию:
``fwrite(&student, sizeof(student), 1, fp);``

Символьный ввод/вывод и циклы чтения файла

Цикл с проверкой EOF:
````
int c;
while ((c = fgetc(fp)) != EOF) {
putchar(c);
}
````
Потому что fgetc возвращает int (чтобы вместить EOF = –1 и все значения unsigned char).

Если заранее известно количество элементов для чтения или если формат файла
позволяет определить конец данных другим способом (например, нулевой символ в
строках), цикл чтения можно организовать без проверки EOF.

### Функции посимвольного чтения

fgetc(fp), getc(fp), getchar() — возвращают int

fgets(buf, size, fp) — читает строку до '\n' или size-1 символов

fputs, fputc, putc, putchar

### Выделение лексем из строки. Принцип границы и strtok
strtok — классическая, но ОПАСНАЯ функция (не потокобезопасна, модифицирует исходную строку).

#### Принцип работы:

Первый вызов strtok(str, delim) — ищет первый токен в str, заменяет разделитель на '\0', возвращает указатель

Последующие вызовы strtok(NULL, delim) — продолжают поиск с места остановки

Внутри использует статическую переменную → нельзя использовать одновременно на двух строках

Правильный цикл:
````
char s[] = "apple,orange;banana grape";
char *token = strtok(s, " ,;");
while (token != NULL) {
    printf("%s\n", token);
    token = strtok(NULL, " ,;");
}
````

Альтернатива без strtok:
````
char *my_strtok(char *str, const char *delim) {
    static char *pos = NULL;
    если str != NULL → pos = str;
    если pos == NULL → возврат NULL;

    // пропускаем разделители
    while (*pos && strchr(delim, *pos)) pos++;

    если (*pos == '\0') → возврат NULL;

    char *start = pos;
    while (*pos && !strchr(delim, *pos)) pos++;

    если (*pos) { *pos++ = '\0'; }  // ставим границу

    return start;
}
````
