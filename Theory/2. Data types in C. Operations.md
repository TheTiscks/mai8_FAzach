## Типы данных в языке C. Операции над переменными стандартных типов  и их особенности. Операция приведения типов. Арифметические,  логические и бинарные операции. Операция быстрого возведения в степень, её алгоритмическая сложность.

В языке C все переменные имеют строго определённый тип, размер и представление в памяти. Это напрямую влияет на корректность и эффективность алгоритмов.
Основные стандартные типы (и их типичные размеры на 64-битных системах):


#### Целочисленные типы

○ char: символьный тип, обычно занимает 1 байт (-128 до 127 или 0 до
255).

○ short (или short int): короткое целое, обычно занимает 2 байта.

○ int: целое число, обычно занимает 4 байта.

○ long (или long int): длинное целое, обычно занимает 4 или 8 байт.

○ long long (или long long int): очень длинное целое, обычно занимает 8
байт. ○ unsigned: модификатор, который делает целочисленные типы
беззнаковыми (только положительные значения).


#### Типы с плавающей точкой

○ float: одинарной точности, обычно занимает 4 байта.
○ double: двойной точности, обычно занимает 8 байт.
○ long double: повышенной точности, обычно занимает 10 или 16 байт.

#### Тип void

○ void: обозначает отсутствие типа, используется для функций, которые не
возвращают значение, или для указателей на неопределенный тип


Модификаторы: signed / unsigned, short, long.


#### Арифметические операции: +, –, *, /, %
Особенности:

* Деление целых — целочисленное с отбрасыванием дробной части (5/2 = 2).
* Деление на ноль — неопределённое поведение (UB).
* Переполнение signed целых — неопределённое поведение.
* Переполнение unsigned — определённое (модуль 2ᵏ).


### Логические операции: &&, ||, !
Результат всегда 0 или 1 (тип int).
Короткое замыкание: правый операнд не вычисляется, если результат уже ясен.


#### Побитовые (бинарные) операции: & | ^ ~ << >>
Работают с битами целых типов.
Сдвиги на unsigned — определённое поведение.
Сдвиг signed влево на отрицательное значение или на количество бит ≥ размера типа — UB.


### Операция приведения типов
Явное: (тип)выражение

Неявное (обычное арифметическое преобразование):
При смешении типов в выражении происходит повышение по следующему правилу (упрощённо):
char → short → int → unsigned → long → unsigned long → long long → float → double

Пример:
int a = 10;
double b = 3.7;
auto c = a + b; // c имеет тип double

### Быстрое возведение в степень (pow для целых чисел)
Стандартной функции pow из <math.h> для целых лучше не пользоваться — она работает с double и медленная.
На практике используют алгоритм «бинарное возведение в степень» (Exponentiation by squaring).
Алгоритм (итеративный вариант):

````
long long binpow_iter(long long a, long long n) {
    long long res = 1;
    while (n > 0) {
        if (n & 1)          // если младший бит = 1, чётн
            res *= a;
        a *= a;             // переходим к следующему биту
        n >>= 1;    // отсекаем ласт бит
    }
    return res;
}
````

### Сложность алгоритма:
Количество итераций = количество бит в b ≈ ⌊log₂ b⌋ + 1
На каждой итерации — одна или две операции умножения.
Итого: O(log b)

Это один из самых частых вопросов на зачётах: «Напишите быстрое возведение в степень и докажите сложность».

Сравнение: наивный цикл for(i=0; i<b; i++) res*=a; даёт O(b), что неприемлемо при b до 10¹⁸.
