## Динамическое выделение памяти. Функции malloc, calloc, realloc, free. Оператор sizeof.

В языке C размер всех обычных объектов (локальных, глобальных, static) должен быть известен компилятору во время компиляции.
Но в реальных программах часто неизвестно заранее, сколько памяти потребуется (длина строки из файла, количество элементов списка, размер графа и т.д.).

Решение — выделять память во время выполнения из специальной области — кучи (heap).
Это и есть динамическое выделение памяти.

Четыре основные функции из <stdlib.h> — теория и смысл
#### malloc(size_t size)
Запрашивает у операционной системы непрерывный блок размером size байтов.
Возвращает указатель на начало блока или NULL, если памяти не хватило.
Выделенная память не инициализирована — содержит случайные значения («мусор»).
#### calloc(size_t count, size_t size)
Делает то же, что malloc(count × size), но дополнительно заполняет весь блок нулями.
Используется, когда нужно гарантировать нулевую инициализацию (массивы, структуры).

#### realloc(void *ptr, size_t new_size)
Изменяет размер ранее выделенного блока.
* если рядом есть свободное место — просто расширяет блок на месте
* если нет — выделяет новый блок нужного размера, копирует туда старые данные, освобождает старый блок
* если ptr == NULL — ведёт себя как malloc
* если new_size == 0 — ведёт себя как free
Возвращает указатель на новый блок (может отличаться от старого!) или NULL при ошибке.

#### free(void *ptr)
Возвращает ранее выделенный блок обратно в кучу.
После вызова указатель становится недействительным (dangling pointer).
Повторный free одного и того же блока — неопределённое поведение.
free(NULL) — безопасно, ничего не делает.

### Оператор sizeof — теория
Это оператор (не функция!) времени компиляции.
Возвращает количество байтов, которое занимает объект или тип (результат имеет тип size_t).

Работает с типами: sizeof(int), sizeof(double), sizeof(struct Node)

Работает с выражениями: sizeof *ptr — размер объекта, на который указывает ptr

Не вычисляет выражение: sizeof(a++) — значение a не изменится

Для массива, объявленного в текущей области видимости, sizeof(arr) даёт полный размер массива в байтах

При передаче массива в функцию он «превращается» в указатель → sizeof даст размер указателя (4 или 8 байт)
