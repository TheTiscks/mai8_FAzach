### Условный оператор if и его формы

Простой if — выполняет блок, если условие ≠ 0

if-else — выбирает один из двух блоков

Цепочка else if … else if … else — последовательная проверка условий

Вложенные if — любой уровень вложенности, но при глубине > 3–4 читаемость падает

В условии любое выражение автоматически приводится к типу int: 0 ложь, всё остальное истина.


### Логические операции

! — логическое НЕ — нет короткого замыкания — самый высокий приоритет

&& — логическое И — есть короткое замыкание (правое не вычисляется, если левое 0) — средний приоритет

|| — логическое ИЛИ — есть короткое замыкание (правое не вычисляется, если левое ≠ 0) — самый низкий приоритет среди логических

Результат всегда 0 или 1.

Классическая идиома безопасного доступа через указатель:
if (ptr && ptr->value > 10) …


#### Тернарный оператор ?: — единственный тернарный оператор в C
Синтаксис: условие ? выражение_если_истина : выражение_если_ложь

Вычисляется только одно из двух выражений (то, которое выбрано)
Приоритет ниже, чем у && и ||, но выше присваивания

Результат имеет общий тип обоих выражений после обычных арифметических преобразований

Полезные примеры
int max = (a > b) ? a : b;
int abs = x >= 0 ? x : -x;
printf("Результат: %d\n", ptr ? ptr->data : -1);
Типичные ошибки

Забыть скобки: flag ? a = 5 : b = 10; → синтаксическая ошибка
Вложенные тернарные операторы без скобок: a ? b : c ? d : e → читается как a ? b : (c ? d : e)
Возврат разных типов без приведения (до C99 запрещено, в современном C допускается с предупреждениями)
