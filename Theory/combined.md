## Алгоритмическая сложность. «О большое». Худший\лучший\средний случай


Алгоритмическая сложность — это характеристика, которая описывает, как много ресурсов (времени или памяти) требуется алгоритму для решения задачи в зависимости от размера входных данных. Она позволяет сравнивать эффективность различных алгоритмов и выбирать наиболее подходящий для конкретной ситуации.

«О большое» (Big O notation) — это математическая нотация, которая используется для описания асимптотического поведения функции. В контексте алгоритмической сложности, она описывает, как время выполнения или объем используемой памяти алгоритма растет с увеличением размера входных данных. 
«О большое» отбрасывает константы и младшие члены, фокусируясь на доминирующем факторе роста.


Основные классы сложности (в порядке возрастания):

● O(1) - константное время: время выполнения не зависит от размера входных
данных (например, доступ к элементу массива по индексу).

● O(log n) - логарифмическое время: время выполнения растет пропорционально
логарифму размера входных данных (например, бинарный поиск).

● O(n) - линейное время: время выполнения растет пропорционально размеру
входных данных (например, линейный поиск).

● O(n log n) - линейно-логарифмическое время: время выполнения растет
пропорционально произведению размера входных данных на его логарифм
(например, быстрая сортировка, сортировка слиянием).

● O(n^2) - квадратичное время: время выполнения растет пропорционально
квадрату размера входных данных (например, сортировка пузырьком,
сортировка вставками).

● O(n^3) - кубическое время: время выполнения растет пропорционально кубу
размера входных данных (например, перемножение матриц классическим
способом).

● O(2^n) - экспоненциальное время: время выполнения растет экспоненциально с
увеличением размера входных данных (например, задача о рюкзаке).

● O(n!) - факториальное время: время выполнения растет пропорционально
факториалу размера входных данных (например, задача коммивояжера).



Худший, лучший и средний случай:
● Худший случай (Worst case): Оценка сложности алгоритма для наихудшего
возможного набора входных данных. Этот показатель гарантирует, что алгоритм
не будет работать медленнее, чем указанная сложность, независимо от
входных данных.

● Лучший случай (Best case): Оценка сложности алгоритма для наилучшего
возможного набора входных данных. Этот показатель обычно не так важен, так
как он не дает представления о поведении алгоритма в большинстве случаев.

● Средний случай (Average case): Оценка сложности алгоритма, усредненная по
всем возможным наборам входных данных. Этот показатель наиболее
реалистичен, но его сложность заключается в том, что необходимо знать
вероятностное распределение входных данных.


Пример:

Рассмотрим алгоритм линейного поиска в массиве.

● Худший случай: Искомый элемент находится в конце массива или отсутствует. В
этом случае потребуется проверить все n элементов, сложность O(n).
● Лучший случай: Искомый элемент находится в начале массива. В этом случае
потребуется всего одна проверка, сложность O(1).
● Средний случай: Если предположить, что искомый элемент может быть в любой
позиции массива с равной вероятностью, то в среднем потребуется проверить

### Функции с переменным числом аргументов (variadic functions)
Объявление:
````
void func(int n, ...);        // хотя бы один фиксированный параметр обязателен
int printf(const char *fmt, ...);
````
Основные типы и макросы из <stdarg.h> (работают только внутри функции с …):

* va_list ap;               — переменная-список аргументов
* va_start(ap, last);       — инициализация; last — имя последнего фиксированного параметра
* va_arg(ap, type)        — извлечь следующий аргумент указанного типа
* va_copy(dest, src)        — скопировать va_list (с C99)
* va_end(ap)                — завершить работу со списком (обязательно)

Правильный шаблон:
````
#include <stdarg.h>
double average(int count, ...) {
    va_list ap;
    va_start(ap, count);
    double sum = 0;
    for (int i = 0; i < count; i++)
        sum += va_arg(ap, double);   // тип указываем вручную!
    va_end(ap);
    return count ? sum / count : 0;
}
````

#### Важно:

* Нет проверки типов и количества — ошибка приводит к UB
* Нельзя пропускать аргументы
* По умолчанию происходит повышение типов: float → double, char/short → int

````

Вызов через указатель:

### Функция atexit

Прототип:
````
int atexit(void (*func)(void));
````
Регистрирует функцию, которая будет вызвана автоматически при нормальном завершении программы (exit(), return из main, конец main).
#### Особенности:
* Можно зарегистрировать до 32 функций (гарантировано стандартом, обычно больше)
* Вызываются в порядке, обратном регистрации (как стек)
* Не вызываются при abort() и при выходе через _Exit()
* Функция не должна принимать параметры и возвращать значение

Пример:
````
void cleanup1(void) { printf("Первый cleanup\n"); }
void cleanup2(void) { printf("Второй cleanup\n"); }

int main() {
atexit(cleanup1);
atexit(cleanup2);        // будет вызван первым
// …
return 0;               // при выходе: cleanup2 → cleanup1
}
````## Динамическое выделение памяти. Функции malloc, calloc, realloc, free. Оператор sizeof.

В языке C размер всех обычных объектов (локальных, глобальных, static) должен быть известен компилятору во время компиляции.
Но в реальных программах часто неизвестно заранее, сколько памяти потребуется (длина строки из файла, количество элементов списка, размер графа и т.д.).

Решение — выделять память во время выполнения из специальной области — кучи (heap).
Это и есть динамическое выделение памяти.

Четыре основные функции из <stdlib.h> — теория и смысл
#### malloc(size_t size)
Запрашивает у операционной системы непрерывный блок размером size байтов.
Возвращает указатель на начало блока или NULL, если памяти не хватило.
Выделенная память не инициализирована — содержит случайные значения («мусор»).
#### calloc(size_t count, size_t size)
Делает то же, что malloc(count × size), но дополнительно заполняет весь блок нулями.
Используется, когда нужно гарантировать нулевую инициализацию (массивы, структуры).

#### realloc(void *ptr, size_t new_size)
Изменяет размер ранее выделенного блока.
* если рядом есть свободное место — просто расширяет блок на месте
* если нет — выделяет новый блок нужного размера, копирует туда старые данные, освобождает старый блок
* если ptr == NULL — ведёт себя как malloc
* если new_size == 0 — ведёт себя как free
Возвращает указатель на новый блок (может отличаться от старого!) или NULL при ошибке.

#### free(void *ptr)
Возвращает ранее выделенный блок обратно в кучу.
После вызова указатель становится недействительным (dangling pointer).
Повторный free одного и того же блока — неопределённое поведение.
free(NULL) — безопасно, ничего не делает.

### Оператор sizeof — теория
Это оператор (не функция!) времени компиляции.
Возвращает количество байтов, которое занимает объект или тип (результат имеет тип size_t).

Работает с типами: sizeof(int), sizeof(double), sizeof(struct Node)

Работает с выражениями: sizeof *ptr — размер объекта, на который указывает ptr

Не вычисляет выражение: sizeof(a++) — значение a не изменится

Для массива, объявленного в текущей области видимости, sizeof(arr) даёт полный размер массива в байтах

### Структура (struct)
Структура — это пользовательский тип данных, который объединяет несколько переменных (членов) разных типов под одним именем.
Члены хранятся в памяти последовательно (с возможным выравниванием — padding).

#### Определение:
Объявление типа: struct Tag { int a; double b; char c; };
Создание объекта: struct Tag obj;
Анонимная структура (редко): struct { ... } obj;

Размер структуры
sizeof(struct Tag) ≥ сумма размеров членов.

Компилятор добавляет padding для выравнивания членов по границам, соответствующим их типу (обычно по 4 или 8 байтам на x64).

Последний член может иметь padding до размера самого большого члена (для массивов структур).

Чтобы минимизировать padding — располагать члены от большего типа к меньшему.

#### Особенности использования

* Доступ к члену: оператор «.» (точка) — obj.member
* Можно копировать структуры присваиванием: struct1 = struct2; (побитовое копирование)
* Массив структур хранится плотно (с учётом padding одного элемента)
* Гибкий член массива (flexible array member, с C99): последний член char data[]; — размер структуры не включает его, выделяется динамически.

### Объединение (union)
Объединение — тип данных, где все члены занимают одну и ту же область памяти (начиная с одного адреса).

Размер union равен размеру самого большого члена (плюс возможный padding до выравнивания).

Смысл: экономия памяти или интерпретация одних и тех же байтов разными типами (например, типизированный вариант void*).

Определение аналогично struct, но ключевое слово union.

Доступ к члену тоже через «.»

В любой момент времени «активен» только последний записанный член — чтение другого даёт неопределённое поведение (но на практике работает для low-level).

### Указатели на структуры

Указатель на структуру объявляется как struct Tag *p;
* Можно указывать на объект: p = &obj;
* Или на динамически выделенную: p = malloc(sizeof(struct Tag));

### Оператор «->»
Сокращение для доступа к члену через указатель: p->member

Эквивалентно (*p).member

Порядок работы препроцессора
Препроцессор — это первый этап компиляции (перед собственно компилятором).

Он обрабатывает исходный текст программы последовательно сверху вниз и выполняет следующие действия:

1. Удаление комментариев: Заменяет все комментарии в коде на пробелы.
2. Обработка директив: Выполняет директивы препроцессора.
3. Макроподстановка: Заменяет все макросы в коде на их значения.
4. Подключение заголовочных файлов: Вставляет содержимое заголовочных
   файлов в исходный код.
5. Условная компиляция: Включает или исключает определенные участки кода в
   зависимости от условий
Результат — «чистый» текст на C, который передаётся компилятору.
Препроцессор не знает синтаксис C — работает чисто текстуально.

### #ifdef / #ifndef / #endif
Условная компиляция.

#ifdef NAME — если макрос NAME определён

#ifndef NAME — если макрос NAME НЕ определён

#else и #elif — аналогично if-else

Используется для:
* защита от повторного включения заголовка
* платформо-зависимого кода
* отладочных вставок (#ifdef DEBUG ... #endif)

### # и ## — операторы макроса

##### # — строковая конкатенация (превращает параметр в строковый литерал)
##### ## — конкатенация токенов (склеивает два токена в один)

### Особенности конструирования замещающих значений для макроопределений
Макросы — чисто текстовая замена, без проверки типов и вычисления выражений на этапе определения.
#### Главные проблемы и правила:

* Оборачивать макросы с параметрами в скобки: #define SQR(x) ((x)*(x)) — иначе приоритет операций сломает смысл
* Оборачивать весь макрос в скобки при сложных выражениях
* Избегать побочных эффектов: #define INC(x) (x++) — вызов INC(a++) приведёт к двойному инкременту
* Рекурсия макросов запрещена (кроме специальных случаев)
* Макросы не видят область видимости — заменяются везде после определения
#undef NAME — отменяет определение макроса

Примеры использования

Include guard (самый частый вопрос):
````
#ifndef MYHEADER_H
#define MYHEADER_H
// содержимое заголовка
#endif
````
Макрос с параметрами и #:
````
#define PRINT_VAR(x) printf(#x " = %d\n", x)
PRINT_VAR(count);   // преобразуется в printf("count" " = %d\n", count);
````
Макрос с ## (склейка токенов):
````
#define VAR(name) var_##name
int VAR(1) = 10;    // преобразуется в int var_1 = 10;
````
Условная компиляция:
````
#ifdef DEBUG
printf("Debug: value = %d\n", value);
#endif
````

Методы перераспределения памяти при работе с динамическими массивами, их вычислительные сложности. Операций добавления/удаления элемента, доступа к элементу по индексу и по значению, их вычислительные сложности. Реализация взаимодействия с динамическим массивом на языке C

#### Динамический массив — это структура данных, имитирующая массив переменного размера.
В C нет встроенных векторов (как std::vector в C++), поэтому реализуется вручную через указатель и динамическую память.

#### Основные компоненты структуры
Обычно оформляется как структура:
* указатель на элементы (типа T*)
* текущий размер (size_t size) — сколько элементов хранится
* ёмкость (size_t capacity) — сколько памяти выделено (capacity ≥ size)

### Создание
Выделяем начальную память (например, на 4–8 элементов):
````C
typedef struct {
    int *data;
    size_t size;
    size_t capacity;
} DynArray;

DynArray* create_array(size_t initial_capacity) {
    DynArray *arr = malloc(sizeof(DynArray));
    arr->data = malloc(initial_capacity * sizeof(int));
    arr->size = 0;
    arr->capacity = initial_capacity;
    return arr;
}
````

#### Перераспределение памяти 
Когда size == capacity, нужно увеличить ёмкость. Для этого используется
функция realloc.
#### realloc:
● Изменяет размер ранее выделенного блока памяти.

● Принимает указатель на старый блок памяти и новый размер в байтах.

● Возвращает указатель на новый блок памяти (возможно, перемещенный в
другое место) или NULL в случае ошибки.

● Содержимое старого блока памяти до нового размера копируется в новый блок

* Стандартная стратегия — умножение на коэффициент роста (обычно 1.5 или 2).
* Сложность: амортизированная O(1) при добавлении, худшая O(n) при одном перераспределении.
### Основные операции и их вычислительная сложность
* Доступ по индексу
``arr->data[index]`` — O(1)

Проверка границ обычно на пользователе (или вручную).

#### Добавление элемента в конец (push_back) 

* * Если size < capacity — просто arr->data[size++] = value; O(1)

* * Если size == capacity — realloc на новую ёмкость, копирование старых данных — O(n)
* Амортизированная сложность при последовательном добавлении n элементов: O(1) на операцию.

#### Удаление элемента из конца (pop_back) - O(1)
````
arr->size--; 
````

Опционально уменьшать capacity (редко, чтобы избежать частых realloc).

#### Удаление элемента по индексу
Сдвиг всех последующих элементов влево — O(n)
(аналогично memmove).

#### Поиск элемента по значению
Линейный поиск — O(n) в худшем и среднем случае.

Вычислительные сложности операций доступа к элементу (по индексу/значению), вставки/удаления элементов для списков соответствующего типа, их реализация на языке C

Списочные структуры данных представляют собой последовательность элементов, где
каждый элемент хранит данные и указатель (или указатели) на следующий (и предыдущий в случае двусвязного списка) элемент.


Односвязный список (singly linked list) — это структура данных, состоящая из узлов, где
каждый узел содержит данные и указатель на следующий узел в списке. Последний
узел списка указывает на NULL.

Структура узла:
````C
typedef struct Node {
    int data;
    struct Node *next;
} Node;
````

Операции и сложности:

* Доступ по индексу — O(n) (проходим по цепочке)
* Поиск по значению — O(n) в худшем случае
* Вставка в начало — O(1)
* Вставка в конец — O(n) (если нет указателя на хвост) или O(1) (если есть)
* Вставка после заданного узла — O(1)
* Удаление первого элемента — O(1)
* Удаление по значению/после узла — O(n) поиск + O(1) удаление

### Двусвязный список (doubly linked list)
Каждый узел содержит данные, указатель на следующий и на предыдущий узел.
Структура узла:
````C
typedef struct DNode {

    struct DNode *prev;

Операции и сложности (по сравнению с односвязным):

* Доступ по индексу — всё ещё O(n)
* Поиск по значению — O(n)
* Вставка/удаление перед/после заданного узла — O(1) (не нужно искать предыдущий)
* Удаление последнего элемента — O(1) (если есть указатель на хвост)

Можно проходить в обе стороны

### Кольцевой список (circular linked list)

Последний узел указывает на первый. Бывает односвязным или двусвязным.

* Нет явного конца → цикл for(;;) с проверкой возврата к head

Часто используется в планировщиках задач (round-robin)

#### Сложности операций (общие для всех типов списков):
* Доступ по индексу — O(n)
* Поиск по значению — O(n)
* Вставка/удаление в начало — O(1)
* Вставка/удаление в конец — O(1) с указателем на хвост, иначе O(n)
* Вставка/удаление после заданного узла — O(1)

    new_node->data = value;

    *head = new_node;
}

void pop_front(Node **head) {

    *head = (*head)->next;

}

void delete_after(Node *prev) {

    prev->next = temp->next;
    free(temp);
}
````

## Списочные структуры данных. Очереди и стеки. Организация очереди и стека на основе односвязного списка. Вычислительные сложности операций доступа к элементу, вставки и удаления элементов для стека и очереди, их реализация на языке C. Алгоритм проверки баланса скобок в выражении.

Очереди и стеки — это абстрактные типы данных (АТД), которые представляют собой коллекции элементов, доступ к которым осуществляется по определенным правилам.

### Очереди
Очередь (queue) — это структура данных, работающая по принципу "первым пришел
— первым ушел" (FIFO - First In, First Out). Элементы добавляются в конец очереди
(enqueue) и удаляются из начала (dequeue)

* enqueue(element): Добавляет элемент в конец очереди.
* dequeue(): Удаляет и возвращает элемент из начала очереди.
* front(): Возвращает элемент из начала очереди без удаления.
* isEmpty(): Проверяет, пуста ли очередь.
* size(): Возвращает количество элементов в очереди.

````C
typedef struct {
    Node *head;
    Node *tail;
} Queue;

void queue_enqueue(Queue *q, int value) {
    Node *n = malloc(sizeof(Node));
    n->data = value;
    n->next = NULL;
    if (q->tail == NULL) {
        q->head = q->tail = n;
    } else {
        q->tail->next = n;
        q->tail = n;
    }
}

int queue_dequeue(Queue *q) {
    Node *temp = q->head;
    int value = temp->data;
    q->head = q->head->next;
    if (q->head == NULL) q->tail = NULL;
    free(temp);
    return value;
}
````


### Стек
Стек (stack) — это структура данных, работающая по принципу "последним пришел —
первым ушел" (LIFO - Last In, First Out). Элементы добавляются и удаляются с
вершины стека.

Операции:
* push(element): Добавляет элемент на вершину стека.
* pop(): Удаляет и возвращает элемент с вершины стека.
* peek(): Возвращает элемент с вершины стека без удаления.
* isEmpty(): Проверяет, пуст ли стек.
* size(): Возвращает количество элементов в стеке

````C
typedef struct {
    Node *top;
} Stack;

void stack_push(Stack *s, int value) {
    Node *n = malloc(sizeof(Node));
    n->data = value;
    n->next = s->top;
    s->top = n;
}

int stack_pop(Stack *s) {
    Node *temp = s->top;
    int value = temp->data;
    s->top = temp->next;
    free(temp);
    return value;
}
````


### Операция size для стека и очереди
##### Теория: операция size возвращает количество элементов в структуре данных.
Для стека и очереди на основе односвязного списка сложность O(n), если просто проходить по списку (неэффективно для больших структур).
Чтобы сделать O(1), добавляем счётчик в структуру (size_t count) и обновляем его при каждой вставке/удалении.

Для стека, добавляем в структуру: 
````C
typedef struct {
    Node *top;
    size_t count;    // для O(1) size
} Stack;
````

Инициализация: 
````C
s->count = 0;
push: s->count++;
pop: s->count--;
size: return s->count;   // O(1)
````



#### Для очереди
Аналогично добавляем count в Queue.

````C
enqueue: q->count++;
dequeue: q->count--;
size: return q->count;   // O(1)
````

### Инфиксная форма (infix)
Операторы располагаются между операндами (например, ``2 + 3 × 4``).
Это привычная нам математическая запись.

Недостатки: нужны скобки для задания порядка вычислений, приоритет операций должен учитываться вручную.
### Постфиксная форма (postfix, обратная польская нотация, RPN)
* Операторы стоят после своих операндов (например, 2 3 4 × +).
#### Преимущества:
* скобки не нужны вообще
* однозначный порядок вычислений
* идеально подходит для вычисления на стеке

Пример: ``(2 + 3) × 4 → 2 3 + 4 ×``

### Префиксная форма (prefix, польская нотация, PN)
Операторы стоят перед своими операндами (например, ``+ 2 × 3 4``).

Преимущества те же, что у постфиксной: нет скобок, однозначность.

! Используется реже, чем постфиксная, но применяется в некоторых компиляторах и Lisp-подобных языках.

### Алгоритм преобразования из инфиксной в постфиксную форму (Shunting Yard)
Используем стек для операторов и выходную очередь.
#### Правила:
* Число → сразу в выход
* '(' → в стек
* ')' → выталкиваем из стека в выход всё до '('
* Оператор → пока на вершине стека оператор с приоритетом ≥ текущего — выталкиваем в выход, затем кладём текущий
* В конце — выталкиваем остаток стека в выход

Сложность: O(n)

### Алгоритм вычисления постфиксного выражения
Используем один стек для операндов.

Идём слева направо:
* Число → кладём в стек
* Оператор → берём два верхних числа (правое — второй операнд), выполняем операцию, результат обратно в стек
* В конце в стеке остаётся единственное число — результат.

* Сложность: O(n)

Дерево – конечное множество объектов Т, состоящее из одного или более узлов, для которых выполняются следующие условия:
* имеется один специально выделенный узел, называемый корнем данного дерева;
* остальные узлы (исключая корень) содержатся в m попарно непересекающихся множествах T1, …,Tm, каждое из которых в свою очередь является деревом. Деревья T1, ...,Тm называются поддеревьями данного корня.


Дерево общего вида (general tree), также известное как n-арное дерево, — это
древовидная структура данных, в которой каждый узел может иметь любое количество
дочерних узлов. 

В отличие от бинарного дерева, где каждый узел имеет не более двух
потомков, в общем дереве количество потомков не ограничено

### Основные операции над деревом общего вида
* Создание узла, O(1)
* Добавление потомка к узлу, O(1) - если есть список потомков как связный список | O(n)
* Обход дерева (прямой, обратный, по уровням), O(n)
* ``Обход в глубину (Depth-First Traversal - DFT): Предусматривает
посещение узлов как можно "глубже" в поддереве, прежде чем
переходить к другим ветвям. Существуют разные варианты DFT, такие
как preorder (сначала посещается корень, затем рекурсивно обходятся
поддеревьяю. К-Л-П), inorder (в бинарных деревьях, ЛКП) и postorder (сначала
рекурсивно обходятся поддеревья, затем посещается корень, ЛПК).
Обход в ширину (Breadth-First Traversal - BFT): Предусматривает
посещение узлов уровень за уровнем, начиная с корня. Обычно
реализуется с использованием очереди.``


* Поиск узла по значению, O(n)
* Удаление поддерева, O(n) в худшем случае
* Подсчёт количества узлов, высоты, листьев и т.д., O(n)

````C

## Отношение порядка на пространстве элементов. Кастомизация операции отношения порядка для заданного контекста в языке C. Быстрая сортировка и функция qsort. Алгоритм дихотомического поиска, его алгоритмическая сложность.

Отношение порядка — это правило, по которому элементы можно упорядочить.
Для сортировки важно понимать, что сравнение элементов должно быть:

* Рефлексивным (a ≤ a) — в программировании обычно не используется явно.
* Антисимметричным (если a ≤ b и b ≤ a → a = b)
* Транзитивным (если a ≤ b и b ≤ c → a ≤ c)
* Полным (любые два элемента сравнимы — важно для корректной сортировки)

В программировании это сводится к функции сравнения, которая:
1. принимает два элемента
2. решает, какой «меньше»
3. возвращает отрицательное / ноль / положительное значение

То есть в C отношение порядка задаётся функцией сравнения (компаратором).

### Кастомизация операции отношения порядка в C
В C нет встроенных средств для перегрузки операторов, поэтому "кастомизация порядка" делается с помощью функций, которые мы передаём как указатели.

``int cmp(const void *a, const void *b);`` возвращает < 0 — если a < b; 0 — если a == b, > 0 - если a > b

Примеры:
````C
int cmp_int(const void *a, const void *b) {
    return *(const int*)a - *(const int*)b;
}

int cmp_str(const void *a, const void *b) {
    return strcmp(*(const char* const*)a, *(const char* const*)b);
}

typedef struct { int x, y; } Point;
int cmp_point_by_x(const void *a, const void *b) {
    const Point *p1 = a, *p2 = b;
    return (p1->x > p2->x) - (p1->x < p2->x);  // безопасно от переполнения
}
````

В stdlib.h есть универсальная функция:
````C
void qsort(
    void *base,
    size_t nitems,
    size_t size,
    int (*compar)(const void *, const void *)
);

````

* base — адрес массива
* nitems — количество элементов
* size — размер одного элемента
* compar — указатель на функцию сравнения
  
Алгоритм внутри — быстрая сортировка (quicksort):

* худший случай: O(n²) — уже отсортированный массив + плохой выбор опорного
* средний и лучший случай: O(n log n)
* память: O(log n) - рекурсия

Применяется только к отсортированным массивам.


### Алгоритм дихотомического (бинарного) поиска
Идея:
1. Найти середину.
2. Сравнить искомое значение со значением середины.
3. Если меньше — искать в левой половине.
4. Если больше — в правой.
5. Повторять пока не останется один элемент или не найдём искомое.

#### Сложность
Время: O(log n)
Память: O(1) (итеративная версия)

Прототип в Си:
````C
void *bsearch(const void *key, const void *base,
              size_t nmemb, size_t size,
              int (*compar)(const void *, const void *));
````

При смешении типов в выражении происходит повышение по следующему правилу (упрощённо):

Быстрое возведение в степень (pow для целых чисел)
Стандартной функции pow из <math.h> для целых лучше не пользоваться — она работает с double и медленная.
На практике используют алгоритм «бинарное возведение в степень» (Exponentiation by squaring).
Алгоритм (итеративный вариант):

````
long long binpow_iter(long long a, long long n) {
    long long res = 1;
    while (n > 0) {
        if (n & 1)          // если младший бит = 1, чётн
            res *= a;
        a *= a;             // переходим к следующему биту
        n >>= 1;    // отсекаем ласт бит
    }
    return res;
}
````

### Сложность алгоритма:
Количество итераций = количество бит в b ≈ ⌊log₂ b⌋ + 1
На каждой итерации — одна или две операции умножения.
Итого: O(log b)

### Обход дерева: Существуют различные способы обхода дерева, включая:
○ Обход в глубину (Depth-First Traversal - DFT): Предусматривает
посещение узлов как можно "глубже" в поддереве, прежде чем
переходить к другим ветвям.

* inorderTraversal — обход узлов в отсортированном порядке, ЛКП
* preorderTraversal — обход узлов в порядке: вершина, левое поддерево, правое поддерево,
* postorderTraversal — обход узлов в порядке: левое поддерево, правое поддерево, вершина.

○ Обход в ширину (Breadth-First Traversal - BFT): Предусматривает
посещение узлов уровень за уровнем, начиная с корня. Обычно
реализуется с использованием очереди.

#### Восстановление свойств кучи
В случае, если свойства кучи были нарушены для 1 элемента, мы можем восстановить их:

Начинаем с корня, сравниваем ключ и идём влево/вправо.
* Сложность: O(h), где h — высота дерева. Средний случай: O(log n), худший: O(n).
* Связь с дихотомическим поиском: в сбалансированном BST поиск эквивалентен бинарному поиску в отсортированном массиве.
#### Вставка элемента
Идём как при поиске до NULL и создаём новый узел.
Сложность: O(h) → средний O(log n), худший O(n).

* Нет детей — просто удаляем
* Один ребёнок — заменяем узел на ребёнка
* Два ребёнка — находим минимальный в правом поддереве (или максимальный в левом), копируем его ключ, удаляем этот минимальный узел
* Сложность: O(h) → средний O(log n), худший O(n).

#### Создание дерева 
— пустое дерево: root = NULL. O(1)

Наивным способом создания кучи является поочередное добавление всех ее элементов, но данный подход имеет сложность O(Nlog2(N)).
Более рациональный подход имеет сложность O(N) и заключается в том, что мы уже для имеющегося массива начинаем производить восстановление свойств кучи, начиная с предпоследнего слоя (так как последний слой по умолчанию удовлетворяет свойствам кучи) и до корня кучи (нулевого элемента массива).

#### Удаление всего дерева 
Рекурсивный post-order: освободить левое, правое, затем узел. Сложность O(n).

#### Слияние

Вариант с последовательным добавлением элементов II кучи в I имеет сложность O(mlog(n+m)).
Подход со сложностью O(n + m) заключается в добавлении в конец массива, содержащего I кучу, массива, содержащего II кучу и вызов процедуры восстановления свойств кучи, начиная с (n+m)/2 элемента, до корня кучи.

Node* create_node(int key) {
    Node* node = malloc(sizeof(Node));
    node->key = key;
    node->left = node->right = NULL;
    return node;
}

Node* insert(Node* root, int key) {
    if (root == NULL) return create_node(key);
    if (key < root->key)
        root->left = insert(root->left, key);
    else if (key > root->key)
        root->right = insert(root->right, key);
    return root;
}

Node* find(Node* root, int key) {
    if (root == NULL || root->key == key) return root;
    if (key < root->key) return find(root->left, key);
    return find(root->right, key);
}

Node* find_min(Node* root) {
    while (root && root->left) root = root->left;
    return root;
}

Node* delete(Node* root, int key) {
    if (root == NULL) return root;
    if (key < root->key)
        root->left = delete(root->left, key);
    else if (key > root->key)
        root->right = delete(root->right, key);
    else {  // нашли
        if (root->left == NULL) {
            Node* temp = root->right;
            free(root);
            return temp;
        } else if (root->right == NULL) {
            Node* temp = root->left;
            free(root);
            return temp;
        }
        // два ребёнка
        Node* temp = find_min(root->right);
        root->key = temp->key;
        root->right = delete(root->right, temp->key);
    }
    return root;
}

void inorder(Node* root) {      // отсортированный вывод
    if (root) {
        inorder(root->left);

        inorder(root->right);
    }
}

Декартово дерево (treap) – хранит пары (Х, Y) таким образом, что оно является BST по X и heap по Y


Каждый узел имеет два поля:

key — для свойства BST (левое < узел < правое)

Все операции — ожидаемая O(log n), худшая O(n).

### Создание

Создание: все элементы в кучу – по Y, а далее просто вставляем их в дерево по Х.

Отсортируем все n узлов по убыванию приоритета.

Сложность сортировки: O(n log n).

Пусть после сортировки первый узел (с максимальным приоритетом) — y_k.

Делаем y_k корнем всего дерева.

Рекурсивно строим левое и правое поддеревья:

* Все узлы с key < y_k.key → идут в левое поддерево
* Все узлы с key > y_k.key → идут в правое поддерево
* При этом порядок приоритетов в подмассивах уже отсортирован по убыванию (потому что мы взяли подпоследовательность из отсортированного списка).


#### Поиск элемента
Обычный BST-поиск по key.
* Связь с дихотомическим поиском: идентичен бинарному поиску в отсортированном массиве при сбалансированном дереве.
* Сложность: ожидаемая O(log n).
#### Вставка элемента
Вставляем как в обычное BST по key (до листа).

Выполняем повороты, чтобы восстановить свойство кучи по priority.

Повороты:
* left_rotate (если правый ребёнок имеет больший приоритет)
* right_rotate (если левый ребёнок имеет больший приоритет)

#### Удаление элемента
1. Находим узел по key.
2. Поворачиваем его вниз, пока он не станет листом (нарушаем кучу, чтобы дети имели больший приоритет).
3. Удаляем лист.

Удаление всего дерева — рекурсивный post-order, O(log n) до O(n) в худшем случае

### Split
1. Рассмотрим случай, в котором требуется разрезать дерево по ключу, большему ключа корня. Посмотрим, как будут устроены результирующие деревья  T1 и T2:
2. T1 : левое поддерево T1 совпадёт с левым поддеревом T. Для нахождения правого поддерева T1, нужно разрезать правое поддерево T на TR1 и TR2 по ключу k и взять  TR1
3. T2 совпадёт с TR2. 
* Случай, в котором требуется разрезать дерево по ключу, меньше либо равному ключа в корне, рассматривается симметрично.

### Merge
Слить два декартовых дерева в одно. 

Все ключи в первом(левом) дереве должны быть меньше, чем ключи во втором(правом). В результате получается дерево, в котором есть все ключи из первого и второго деревьев:  ``merge(T1,T2)→{T}``.

Пусть нужно слить деревья T1 и T2. Тогда, очевидно, у результирующего дерева T есть корень. Корнем станет вершина из T1 или T2 с наибольшим приоритетом y. Но вершина с самым большим y из всех вершин деревьев T1 и T2 может быть только либо корнем  T1, либо корнем T2.
1. Рассмотрим случай, в котором корень T1 имеет больший y, чем корень T2. Случай, в котором корень T2 имеет больший y, чем корень T1, симметричен этому.
2. Если y корня T1 больше y корня T2, то он и будет являться корнем. Тогда левое поддерево T совпадёт с левым поддеревом T1 . Справа же нужно подвесить объединение правого поддерева T1 и дерева T2.

### Повороты и вставка, код

void right_rotate(Node** y) {
    Node* x = (*y)->left;
    (*y)->left = x->right;
    x->right = *y;
    *y = x;
}

void left_rotate(Node** x) {
    Node* y = (*x)->right;
    (*x)->right = y->left;
    y->left = *x;
    *x = y;
}

void insert(Node** root, int key) {
    if (*root == NULL) {
        *root = create_node(key);
        return;
    }
    if (key < (*root)->key) {
        insert(&(*root)->left, key);
        if ((*root)->left->priority > (*root)->priority)
            right_rotate(root);
    } else if (key > (*root)->key) {
        insert(&(*root)->right, key);
        if ((*root)->right->priority > (*root)->priority)
            left_rotate(root);
    }
}

Приоритетная очередь – это абстрактная структура данных, которая поддерживает
операции добавления элемента и извлечения элемента с наивысшим (или
наименьшим) приоритетом. В отличие от обычной очереди (FIFO) или стека (LIFO),

### Бинарная куча
Бинарная куча (Binary Heap) – это один из способов реализации
приоритетной очереди, использующий структуру данных в виде бинарного дерева.

* Значение в любой вершине не меньше (относительно введенного компаратора), чем значения её потомков. (ДЛЯ MIN-HEAP: НЕ БОЛЬШЕ ЗНАЧ. ПОТОМКОВ)
* На i-ом слое 2i вершин, кроме последнего. Слои нумеруются с нуля.
* Последний слой заполняется слева направо (как показано на рисунке)

Чаще всего бинарную кучу реализуют на динамическом массиве
Хранение на базе массива:
* Left(i) – 2*i + 1
* Right(i) - 2*i+2

````C
typedef struct {
    int *data;      // динамический массив элементов
    size_t size;    // текущее количество элементов
    size_t capacity;// выделенная память
} BinaryHeap;
````

#### Вставка O(log(n))

1. Добавление элемента в конец массива;
2. Сравнение с родителем;
3. Если родитель больше или мы дошли до корня, то вставка окончена;
4. В случае если родитель меньше, то меняемся местами с родителем и возвращаемся к п.2.

````C
void heap_push(MaxHeap *h, int val) {
    if (h->size == h->capacity) {
        h->capacity = h->capacity ? h->capacity * 2 : 4; // нач. емкость 4
        h->data = realloc(h->data, h->capacity * sizeof(int));
    }
    size_t i = h->size++;
    h->data[i] = val;
    while (i > 0 && h->data[i] > h->data[parent(i)]) {
        swap(&h->data[i], &h->data[parent(i)]);
        i = parent(i);
    }
}
````

вспомогательные:
````C
static void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
}

static size_t parent(size_t i) { return (i - 1) / 2; }
static size_t left(size_t i)   { return 2 * i + 1; }
static size_t right(size_t i)  { return 2 * i + 2; }
````

``int heap_max(const MaxHeap *h) {

2. В случае если мы достигли нижней границы кучи (нет потомков), то останавливаемся;
3. Если есть потомки имеющие значение больше чем у нас, повторяем п. 2.

#### Удаление максимума
При удалении максимума мы извлекаем его из кучи и ставим на его место наименьший элемент (последний элемент в массиве)
Далее мы по описанному ранее алгоритму восстанавливаем свойства кучи для корня

O(log n)

````C
int heap_pop(MaxHeap *h) {
    int max_val = h->data[0];
    h->data[0] = h->data[--h->size];
    size_t i = 0;
    while (left(i) < h->size) {
        size_t largest = i;
        size_t l = left(i), r = right(i);
        if (l < h->size && h->data[l] > h->data[largest])
            largest = l;
        if (r < h->size && h->data[r] > h->data[largest])
            largest = r;
        if (largest == i) break;
        swap(&h->data[i], &h->data[largest]);
        i = largest;
    }
    return max_val;
}
````


#### Удаление произвольного элемента O(n)

В классической бинарной приоритетной очереди удаление произвольного элемента не является стандартной операцией, потому что куча не поддерживает быстрый поиск по значению (это не BST).

Аналогично: мы замещаем удаленный элемент минимальным возможным (конец массива) и восстанавливаем для него свойства кучи по описанному ранее алгоритму

#### Слияние куч

##### С разрушением исходных (destructive merge)
* Самый эффективный способ — использовать heap_push последовательно: O((n+m) log(n+m))
* Или построить кучу из объединённого массива за O(n+m) (heapify).

##### Без разрушения исходных (non-destructive merge) 
Создаём новую кучу и вставляем элементы из обеих — O((n+m) log(n+m))


````C
MaxHeap* heap_merge_non_destructive(const MaxHeap *h1, const MaxHeap *h2) {
    MaxHeap *result = malloc(sizeof(MaxHeap));
    result->capacity = h1->size + h2->size;
    result->data = malloc(result->capacity * sizeof(int));
    result->size = 0;
    for (size_t i = 0; i < h1->size; i++)
        heap_push(result, h1->data[i]);
    for (size_t i = 0; i < h2->size; i++)
        heap_push(result, h2->data[i]);
    return result;
}
````

````C
void heap_merge_destructive(MaxHeap *dest, MaxHeap *src) {
    if (dest->size + src->size > dest->capacity) {
        dest->capacity = dest->size + src->size;
        dest->data = realloc(dest->data, dest->capacity * sizeof(int));
    }
    memcpy(dest->data + dest->size, src->data, src->size * sizeof(int));
    dest->size += src->size;
    for (size_t i = dest->size / 2; i != (size_t)-1; i--) {
        size_t j = i;
        while (left(j) < dest->size) {
            size_t largest = j;
            size_t l = left(j), r = right(j);
            if (l < dest->size && dest->data[l] > dest->data[largest])
                largest = l;
            if (r < dest->size && dest->data[r] > dest->data[largest])
                largest = r;
            if (largest == j) break;
            swap(&dest->data[j], &dest->data[largest]);
            j = largest;
        }
    }
}

Левосторонняя куча — это разновидность приоритетной очереди, реализованная в виде бинарного дерева (обычно min-heap), в котором, помимо свойства кучи, выполняется дополнительное левостороннее свойство:
* ! У каждого узла расстояние до ближайшего листа в левом поддереве (npl — null path length) не меньше, чем в правом.
Это гарантирует, что правое поддерево всегда «легче», и слияние работает эффективно.

### Слияние 

* Данная операция является ключевой в схеме работы левостороннего дерева и через нее выражаются все остальные операции.
* Во время слияния двух левосторонних деревьев мы должны поддерживать их основное свойство
* сравниваем корни обоих деревьев, меньший становится новым корнем, а большее дерево сливается с правым поддеревом меньшего 


* Алгоритм:
1. Ориентируясь на то, что дерево левостороннее рекурсивно начинаем слияние нового дерева с правым поддеревом;
2. Если “левостороннесть” кучи была нарушена, меняем получившегося правого и левого ребенка местами;
3. Пересчитываем расстояние до ближайшей свободной позиции

### Вставка
1. Обозначаем новый элемент как левостороннее дерево с 1 вершиной;
2. Производим слияние имеющегося дерева с вновь созданным.

* Сложность: O(log(n))

### Извлечение минимального элемента
1. Извлекаем текущий корень дерева;
2. Производим слияние левого и правого поддеревьев.

* Сложность: O(log(n))

### Удаление элемента
Алгоритм:

1. Найти узел (O(log n) в худшем)
2. Удалить его как минимум (повернуть вверх по приоритету или использовать другой трюк)

! удаление произвольного элемента не считается стандартной операцией leftist heap (в отличие от Fibonacci heap).

* Сложность: O(log(n))

### Построение кучи
Левосторонняя куча строится последовательным добавлением в нее всех необходимых элементов;

* Сложность: O(n) аморт. (через попарные слияния)

````C
#include <stdio.h>
#include <stdlib.h>

// Узел левосторонней кучи (min-heap)
typedef struct Node {
    int key;                   // значение (приоритет)
    int npl;                   // null path length
    struct Node *left;
    struct Node *right;
} Node;

// Представление приоритетной очереди — указатель на корень
typedef Node* LeftistHeap;

// Вспомогательная функция: npl(NULL) = -1
static int get_npl(const Node *t) {
    return t ? t->npl : -1;
}

// Ключевая операция: слияние двух левосторонних куч (с разрушением исходных)
static Node* merge(LeftistHeap h1, LeftistHeap h2) {
    if (!h1) return h2;
    if (!h2) return h1;

    // Обеспечиваем, что h1->key ≤ h2->key
    if (h1->key > h2->key) {
        Node *temp = h1;
        h1 = h2;
        h2 = temp;
    }

    // Рекурсивно сливаем h2 с правым поддеревом h1
    h1->right = merge(h1->right, h2);

    // Восстанавливаем левостороннее свойство
    if (get_npl(h1->left) < get_npl(h1->right)) {
        Node *temp = h1->left;
        h1->left = h1->right;
        h1->right = temp;
    }

    // Пересчитываем npl
    h1->npl = get_npl(h1->right) + 1;

    return h1;
}

// Поиск минимума — O(1)
int find_min(LeftistHeap h) {
    if (!h) {
        fprintf(stderr, "Heap is empty\n");
        exit(1);
    }
    return h->key;
}

// Вставка элемента — O(log n)
LeftistHeap insert(LeftistHeap h, int key) {
    Node *new_node = malloc(sizeof(Node));
    new_node->key = key;
    new_node->npl = 0;
    new_node->left = new_node->right = NULL;
    return merge(h, new_node);
}

// Удаление минимума — O(log n)
int delete_min(LeftistHeap *h) {
    if (!*h) {
        fprintf(stderr, "Heap is empty\n");
        exit(1);
    }
    int min_val = (*h)->key;
    Node *left = (*h)->left;
    Node *right = (*h)->right;
    free(*h);
    *h = merge(left, right);
    return min_val;
}

// Освобождение всей кучи (post-order)
void free_heap(LeftistHeap h) {
    if (h) {
        free_heap(h->left);
        free_heap(h->right);
        free(h);
    }
}


// копирование не разрушая
// Вспомогательная функция — копирование узла
static Node* copy_node(const Node *src) {
    if (!src) return NULL;
    Node *new = malloc(sizeof(Node));
    new->key = src->key;
    new->npl = src->npl;
    new->left = src->left;    // сначала ссылаемся на исходные поддеревья
    new->right = src->right;
    return new;
}

Node* merge_non_destructive(Node *h1, Node *h2) {
    if (!h1) return copy_node(h2);  // если один пуст — копируем другой
    if (!h2) return copy_node(h1);
    Node *root;
    if (h1->key <= h2->key) {
        root = copy_node(h1);
        root->right = merge_non_destructive(h1->right, h2);
    } else {
        root = copy_node(h2);
        root->right = merge_non_destructive(h1, h2->right);
    }
    // Восстанавливаем левостороннее свойство
    if (get_npl(root->left) < get_npl(root->right)) {
        Node *temp = root->left;
        root->left = root->right;
        root->right = temp;
    }
    // Пересчитываем npl
    root->npl = get_npl(root->right) + 1;
    return root;
}
````## Приоритетные очереди. Косая приоритетная очередь, её реализация на базе структуры данных вида бинарное дерево. Операции вставки элемента, поиска минимума/максимума, удаления минимума/максимума, слияния с разрушением и без разрушения исходных косых приоритетных очередей; их вычислительные сложности и реализация на языке C.

Косая куча — это саморегулирующаяся бинарная куча (обычно min-heap), в которой слияние является ключевой операцией.

В отличие от leftist heap, здесь нет дополнительного поля npl.

Саморегуляция достигается за счёт безусловной перестановки левого и правого ребёнка при каждом слиянии.

Это даёт амортизированную сложность O(log n) для всех операций (не гарантированную, как в leftist, но очень хорошую на практике).

•Обладает такими же свойствами, как и левосторонняя куча, но имеет отличие в процессе слияния двух куч.

````C
typedef struct Node {
    int key;                   // значение (приоритет)
    struct Node *left;
    struct Node *right;
} Node;

typedef Node* SkewHeap;        // очередь — указатель на корень
````

Операции и вычислительные сложности

* Поиск минимума/максимума — O(1) (корень кучи)
* Вставка элемента — O(log n) амортизированная (создать узел + слияние)
* Удаление минимума/максимума — O(log n) амортизированная (удалить корень + слияние поддеревьев)
* Слияние двух куч — O(log n) амортизированная. Схоже с алгоритмом для левосторонней кучи, но перестановка левого и правого поддерева БЕЗУСЛОВНА
  
* (
1. Если одна из куч пуста — возвращаем другую.
2. Сравниваем корни двух куч. Если корень второй кучи меньше — меняем кучи местами (чтобы корень результата имел минимальный ключ).
3. Корнем результата становится корень первой кучи.
4. Безусловно меняем местами левое и правое поддеревья этого корня (это ключевое действие).
5. Рекурсивно сливаем вторую кучу с тем поддеревом, которое теперь стало правым (после перестановки).
6. Присваиваем результат рекурсии правому ребёнку корня.
7. Возвращаем новый корень.
  )

Реализация на Си

````C
#include <stdlib.h>

// Ключевая операция: слияние с разрушением (амортизированная O(log n))
SkewHeap merge(SkewHeap h1, SkewHeap h2) {
    if (!h1) return h2;
    if (!h2) return h1;

    // Обеспечиваем h1->key ≤ h2->key
    if (h1->key > h2->key) {
        SkewHeap temp = h1;
        h1 = h2;
        h2 = temp;
    }

    // Безусловная перестановка детей
    SkewHeap temp = h1->left;
    h1->left = h1->right;
    h1->right = temp;

    // Рекурсивно сливаем бывшее правое поддерево с h2
    h1->right = merge(h1->right, h2);

    return h1;
}

// Поиск минимума (min-heap)
int find_min(SkewHeap h) {
    if (!h) {
        // ошибка: пустая куча
        exit(1);
    }
    return h->key;
}

// Вставка
SkewHeap insert(SkewHeap h, int key) {
    SkewHeap new_node = malloc(sizeof(Node));
    new_node->key = key;
    new_node->left = new_node->right = NULL;
    return merge(h, new_node);
}

// Удаление минимума
int delete_min(SkewHeap *h) {
    if (!*h) {
        // ошибка
        exit(1);
    }
    int min_val = (*h)->key;
    SkewHeap left = (*h)->left;
    SkewHeap right = (*h)->right;
    free(*h);
    *h = merge(left, right);
    return min_val;
}

// Слияние без разрушения исходных (копируем путь слияния)
SkewHeap merge_non_destructive(SkewHeap h1, SkewHeap h2) {
    if (!h1) return h2 ? h2 : NULL;  // простая копия, если один пуст
    if (!h2) return h1;

    SkewHeap root;
    if (h1->key <= h2->key) {
        root = malloc(sizeof(Node));
        root->key = h1->key;
        root->left = h1->left;
        root->right = merge_non_destructive(h1->right, h2);
    } else {
        root = malloc(sizeof(Node));
        root->key = h2->key;
        root->left = h2->left;
        root->right = merge_non_destructive(h1, h2->right);
    }

    // Безусловная перестановка
    SkewHeap temp = root->left;
    root->left = root->right;
    root->right = temp;

    return root;
}

// Освобождение кучи
void free_skew_heap(SkewHeap h) {
    if (h) {
        free_skew_heap(h->left);
        free_skew_heap(h->right);
        free(h);
    }
}

Биномиальная и фибоначчиева приоритетные очереди, их реализация на базе структур данных вида кольцевой список и дерево общего вида. ### Биномиальное дерево
Биномиальное дерево ранга k — это дерево, состоящее из корня и k поддеревьев рангов 0, 1, …, k−1.

Число элементов в биномиальном дереве ранга k равно точно 2^k.

Ранг — это количество детей корня (и одновременно логарифм числа узлов).
### Биномиальная куча (binomial heap)
Это лес из биномиальных деревьев разных рангов с разными корневыми ключами, удовлетворяющих свойству кучи (min-heap или max-heap).

Корни деревьев связаны в кольцевой односвязный список (обычно по возрастанию ранга).
### Фибоначчиева куча (Fibonacci heap)
Это тоже лес деревьев общего вида, удовлетворяющих свойству кучи. Деревья связаны в кольцевой двусвязный список корней.

Дополнительные свойства:
* степень узла ограничена O(log n)
* пометка удалённых детей для отложенной очистки

### Операции и их вычислительные сложности:

* Поиск минимума — O(1) для обеих (указатель на минимальный корень или проход по списку корней)
* Вставка — O(log n) для биномиальной, амортизированная O(1) для фибоначчиевой
* Удаление минимума — O(log n) для биномиальной, амортизированная O(log n) для фибоначчиевой
* Слияние с разрушением — O(log n) для биномиальной, амортизированная O(1) для фибоначчиевой
* Слияние без разрушения — O(log n) для обеих (копирование списка корней или пути слияния)

### Алгоритм дихотомического (бинарного) поиска

Дихотомический (бинарный) поиск – это алгоритм для поиска заданного значения в
отсортированном массиве. Он работает по принципу "разделяй и властвуй":
1. Сравниваем искомое значение со средним элементом массива.
2. Если значения равны, возвращаем индекс среднего элемента.3. Если искомое значение меньше среднего элемента, рекурсивно выполняем
   поиск в левой половине массива.
4. Если искомое значение больше среднего элемента, рекурсивно выполняем
   поиск в правой половине массива.

Алгоритмическая сложность: O(log n), где n - количество элементов в массиве. На каждом шаге размер области поиска уменьшается вдвое.

Существует и итеративная версия, реализуемая с циклом while/for. Ее сложность O(1)

## Дерево скобочного выражения. Вычисление арифметических выражений с помощью деревьев. Алгоритм конвертации строкового представления арифметического выражения, записанного в инфиксной форме, в дерево арифметического выражения. Алгоритм вычисления значения арифметического выражения по его дереву.

Дерево скобочного выражения (или дерево арифметического выражения) – это
бинарное дерево, которое представляет арифметическое выражение. Оно позволяет
эффективно вычислять выражения и анализировать их структуру.
* Листья: Операнды (числа, переменные).
* Внутренние узлы: Операторы (+, -, *, /, ^ и т.д.).

### Алгоритм конвертации инфиксной формы в дерево

Существует несколько способов построения дерева из инфиксного выражения. Один
из распространенных подходов – использование алгоритма шунтирующего двора
(Shunting Yard Algorithm) для преобразования инфиксной формы в постфиксную
(обратную польскую запись), а затем построение дерева из постфиксной формы.

Также можно использовать рекурсивный спуск. Рассмотрим второй вариант, как более
наглядный для построения дерева.

#### Рекурсивный спуск:
1. Базовый случай: Если выражение состоит из одного операнда, создать лист с
   этим операндом и вернуть его.
2. Рекурсивный шаг:
   * Найти оператор с наименьшим приоритетом, который не находится
   внутри скобок. Этот оператор станет корнем поддерева.
   * Рекурсивно построить левое поддерево из части выражения слева от
   оператора.
   * Рекурсивно построить правое поддерево из части выражения справа от
   оператора.
3. Dернуть корень поддерева.
Пример: Рассмотрим выражение (2 + 3) * (4 - 1).
1. Находим оператор * с наименьшим приоритетом вне скобок. Он становится
   корнем.
2. Левое поддерево: (2 + 3)
   * Оператор + становится корнем.
   * Левый операнд: 2 (лист)
   * Правый операнд: 3 (лист)
3. Правое поддерево: (4 - 1)
   * Оператор - становится корнем.
   * Левый операнд: 4 (лист)
   * Правый операнд: 1 (лист)
5. Полученное дерево:
````C
*
/ \
+ -
/ \ / \
2 3 4 1
````
### Алгоритм вычисления значения по дереву

Вычисление значения выражения по дереву выполняется рекурсивно:
1. Базовый случай: Если узел – лист, вернуть значение операнда.
2. Рекурсивный шаг:
   * Рекурсивно вычислить значение левого поддерева.
   * Рекурсивно вычислить значение правого поддерева.
   * Выполнить операцию, соответствующую оператору в текущем узле, над
   результатами левого и правого поддеревьев.
3. Вернуть результат операции.
#### Пример (продолжение):
Корень *:
   * Левое поддерево +: 2 + 3 = 5
   * Правое поддерево -: 4 - 1 = 3
   * Результат: 5 * 3 = 15

### Реализация на C (упрощенная, для целых чисел и основных операций):
````C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    char op; // оператор или '0' для числа
    int value; // значение, если op == '0'
    struct Node *left, *right;
} Node;
    // ... (функции для создания узлов, парсинга и построения дерева)
int evaluate(Node* root) {
    if (root->op == '0') {
        return root->value;
}
    int leftValue = evaluate(root->left);
    int rightValue = evaluate(root->right);

    switch (root->op) {
        case '+': return leftValue + rightValue;
        case '-': return leftValue - rightValue;
        case '*': return leftValue * rightValue;
        case '/': return leftValue / rightValue;
        default: return 0; // обработка ошибок
    }
}
````

Хеш-функция — функция, которая по ключу k выдаёт целое число h(k) в диапазоне [0, m−1], где m — размер таблицы.
Используется для быстрого определения «ячейки» таблицы, куда помещать элемент.

#### Свойства хорошей хеш-функции:
* Детерминированность: одинаковый ключ → одинаковый хеш
* Равномерность распределения: ключи равномерно распределяются по 0..m−1
* Быстрота вычисления
* Минимизация коллизий (но полностью избежать их невозможно)

Коллизия — ситуация, когда два разных ключа дают один и тот же хеш: h(k1) = h(k2), k1 ≠ k2.

Метод цепочек — самый распространённый способ разрешения коллизий.
В каждой ячейке таблицы хранится односвязный список (цепочка) элементов с одинаковым хешем.

#### Структура хеш-таблицы с цепочками на C
````C
#define TABLE_SIZE 10007   // простое число для лучшего распределения

typedef struct Entry {
    int key;               // ключ
    int value;             // значение (или void* для универсальности)
    struct Entry *next;
} Entry;

typedef struct {
    Entry *buckets[TABLE_SIZE];
} HashTable;
````

### Операции и их вычислительные сложности
(при хорошей хеш-функции и коэффициенте загрузки α ≈ 1)

* Поиск по ключу

Вычисляем хеш → идём по цепочке в бакете → сравниваем ключи.
Средняя сложность: O(1 + α) ≈ O(1)
Худшая (все в одной цепочке): O(n)

* Добавление элемента

Вычисляем хеш → вставляем в начало цепочки бакета (O(1)).
Средняя: O(1), худшая: O(n)

* Удаление элемента по ключу

Вычисляем хеш → ищем в цепочке → удаляем найденный узел.
Средняя: O(1), худшая: O(n)

### Реализация на C (min-heap, простая версия)
````C
unsigned int hash(int key) {
    unsigned int h = key;
    h = ((h >> 16) ^ h) * 0x45d9f3b;
    h = ((h >> 16) ^ h) * 0x45d9f3b;
    h = (h >> 16) ^ h;
    return h % TABLE_SIZE;
}

void ht_insert(HashTable *ht, int key, int value) {
    unsigned int index = hash(key);
    Entry *new = malloc(sizeof(Entry));
    new->key = key;
    new->value = value;
    new->next = ht->buckets[index];   // вставка в начало цепочки
    ht->buckets[index] = new;
}

int* ht_search(HashTable *ht, int key) {
    unsigned int index = hash(key);
    Entry *cur = ht->buckets[index];
    while (cur) {
        if (cur->key == key) return &cur->value;
        cur = cur->next;
    }
    return NULL;   // не найден
}

void ht_delete(HashTable *ht, int key) {
    unsigned int index = hash(key);
    Entry *cur = ht->buckets[index];
    Entry *prev = NULL;
    while (cur) {
        if (cur->key == key) {
            if (prev) prev->next = cur->next;
            else ht->buckets[index] = cur->next;
            free(cur);
            return;
        }
        prev = cur;
        cur = cur->next;
    }
}

Преобразование строкового представления числа в целочисленное значение является
распространенной задачей. 

Мы рассмотрим общий случай, когда число записано в
системе счисления с основанием base (например, для десятичной системы base = 10,
для двоичной base = 2, для шестнадцатеричной base = 16).

### Схема Горнера
Схема Горнера - это эффективный алгоритм для вычисления значения многочлена:
P(x) = a_n * x^n + a_{n-1} * x^(n-1) + ... + a_1 * x + a_0
В контексте преобразования строки в число, x - это основание системы счисления
(base), а коэффициенты a_i - это цифры числа, представленные в виде символов.
Например, число "123" в десятичной системе можно представить как многочлен 1 *
10^2 + 2 * 10^1 + 3 * 10^0.


### Основная идея схемы Горнера

Любое число в системе счисления с основанием p можно рассматривать как многочлен:

``dₙ₋₁ · pⁿ⁻¹ + dₙ₋₂ · pⁿ⁻² + … + d₁ · p + d₀``

Схема Горнера позволяет вычислить значение этого многочлена за один проход слева направо, используя только умножения и сложения:

``text((…((dₙ₋₁ · p + dₙ₋₂) · p + dₙ₋₃) · p + …) · p + d₀)``

Пример для "735" (p = 10):
→ ((7 × 10 + 3) × 10 + 5) = 735

Всего 2 умножения и 2 сложения вместо вычисления 10² и 10¹ отдельно.

Поэтому преобразование строки → число делается так:
``res = 0``
для каждого символа цифры d в строке слева направо:
``res = res * base + d``
Сложность: O(n) операций, где n — длина строки.

````
long long str_to_ll(const char *s, int base) {
    long long res = 0;
    int sign = 1;

    if (*s == '-') { 
        sign = -1; 
        s++; 
    } else if (*s == '+') s++;
    
    while (*s) {
        int digit;
        if (*s >= '0' && *s <= '9') digit = *s - '0';
        else if (*s >= 'A' && *s <= 'Z') digit = 10 + (*s - 'A');
        else if (*s >= 'a' && *s <= 'z') digit = 10 + (*s - 'a');
        else break; // неверный символ

        if (digit >= base) break; // цифра больше основания

        res = res * base + digit;   // ← это и есть схема Горнера
        s++;
    }
    return sign * res;
}
````


### Обратная схема Горнера
Нужна для преобразования числа → строка в заданной системе счисления.

Идея: последовательно выделяем младшие цифры через остаток от деления.
````
735 ÷ 10 = 73 остаток 5 → последняя цифра
73 ÷ 10 = 7 остаток 3 → следующая
7 ÷ 10 = 0 остаток 7 → первая цифра
Цифры получаются в обратном порядке → сохраняем во временный буфер → разворачиваем.
````
Сложность: O(logₚ |x|) = O(количество цифр в числе).


````
#include <string.h>

void ll_to_str(long long num, char *buf, int base) {
    if (num == 0) {
        strcpy(buf, "0");
        return;
    }

    int sign = 0;
    if (num < 0) {
        sign = 1;
        num = -num;
    }

    char temp[66];  // хватит для long long в двоичной системе
    int i = 0;

    while (num > 0) {
        int digit = num % base;
        if (digit < 10) temp[i++] = '0' + digit;
        else temp[i++] = 'A' + (digit - 10);
        num /= base;
    }

    if (sign) temp[i++] = '-';

    // разворот
    int j = 0;
    while (i > 0) buf[j++] = temp[--i];
    buf[j] = '\0';
}

char *my_strtok(char *str, const char *delim) {
    static char *pos = NULL;
    если str != NULL → pos = str;
    если pos == NULL → возврат NULL;

    // пропускаем разделители
    while (*pos && strchr(delim, *pos)) pos++;

    если (*pos == '\0') → возврат NULL;

    char *start = pos;
    while (*pos && !strchr(delim, *pos)) pos++;

    если (*pos) { *pos++ = '\0'; }  // ставим границу

    return start;

Алгоритмическая сложность — это характеристика, которая описывает, как много ресурсов (времени или памяти) требуется алгоритму для решения задачи в зависимости от размера входных данных. Она позволяет сравнивать эффективность различных алгоритмов и выбирать наиболее подходящий для конкретной ситуации.

«О большое» (Big O notation) — это математическая нотация, которая используется для описания асимптотического поведения функции. В контексте алгоритмической сложности, она описывает, как время выполнения или объем используемой памяти алгоритма растет с увеличением размера входных данных. 
«О большое» отбрасывает константы и младшие члены, фокусируясь на доминирующем факторе роста.


Основные классы сложности (в порядке возрастания):

● O(1) - константное время: время выполнения не зависит от размера входных
данных (например, доступ к элементу массива по индексу).

● O(log n) - логарифмическое время: время выполнения растет пропорционально
логарифму размера входных данных (например, бинарный поиск).

● O(n) - линейное время: время выполнения растет пропорционально размеру
входных данных (например, линейный поиск).

● O(n log n) - линейно-логарифмическое время: время выполнения растет
пропорционально произведению размера входных данных на его логарифм
(например, быстрая сортировка, сортировка слиянием).

● O(n^2) - квадратичное время: время выполнения растет пропорционально
квадрату размера входных данных (например, сортировка пузырьком,
сортировка вставками).

● O(n^3) - кубическое время: время выполнения растет пропорционально кубу
размера входных данных (например, перемножение матриц классическим
способом).

● O(2^n) - экспоненциальное время: время выполнения растет экспоненциально с
увеличением размера входных данных (например, задача о рюкзаке).

● O(n!) - факториальное время: время выполнения растет пропорционально
факториалу размера входных данных (например, задача коммивояжера).



Худший, лучший и средний случай:
● Худший случай (Worst case): Оценка сложности алгоритма для наихудшего
возможного набора входных данных. Этот показатель гарантирует, что алгоритм
не будет работать медленнее, чем указанная сложность, независимо от
входных данных.

● Лучший случай (Best case): Оценка сложности алгоритма для наилучшего
возможного набора входных данных. Этот показатель обычно не так важен, так
как он не дает представления о поведении алгоритма в большинстве случаев.

● Средний случай (Average case): Оценка сложности алгоритма, усредненная по
всем возможным наборам входных данных. Этот показатель наиболее
реалистичен, но его сложность заключается в том, что необходимо знать
вероятностное распределение входных данных.


Пример:

Рассмотрим алгоритм линейного поиска в массиве.

● Худший случай: Искомый элемент находится в конце массива или отсутствует. В
этом случае потребуется проверить все n элементов, сложность O(n).
● Лучший случай: Искомый элемент находится в начале массива. В этом случае
потребуется всего одна проверка, сложность O(1).
● Средний случай: Если предположить, что искомый элемент может быть в любой
позиции массива с равной вероятностью, то в среднем потребуется проверить

### Функции с переменным числом аргументов (variadic functions)
Объявление:
````

Основные типы и макросы из <stdarg.h> (работают только внутри функции с …):

* va_list ap;               — переменная-список аргументов
* va_start(ap, last);       — инициализация; last — имя последнего фиксированного параметра
* va_arg(ap, type)        — извлечь следующий аргумент указанного типа
* va_copy(dest, src)        — скопировать va_list (с C99)
* va_end(ap)                — завершить работу со списком (обязательно)

Правильный шаблон:
````
#include <stdarg.h>
double average(int count, ...) {
    va_list ap;
    va_start(ap, count);
    double sum = 0;
    for (int i = 0; i < count; i++)
        sum += va_arg(ap, double);   // тип указываем вручную!
    va_end(ap);
    return count ? sum / count : 0;
}
````

````## Динамическое выделение памяти. Функции malloc, calloc, realloc, free. Оператор sizeof.

В языке C размер всех обычных объектов (локальных, глобальных, static) должен быть известен компилятору во время компиляции.
Но в реальных программах часто неизвестно заранее, сколько памяти потребуется (длина строки из файла, количество элементов списка, размер графа и т.д.).

Решение — выделять память во время выполнения из специальной области — кучи (heap).
Это и есть динамическое выделение памяти.

Четыре основные функции из <stdlib.h> — теория и смысл
#### malloc(size_t size)
Запрашивает у операционной системы непрерывный блок размером size байтов.
Возвращает указатель на начало блока или NULL, если памяти не хватило.
Выделенная память не инициализирована — содержит случайные значения («мусор»).
#### calloc(size_t count, size_t size)
Делает то же, что malloc(count × size), но дополнительно заполняет весь блок нулями.
Используется, когда нужно гарантировать нулевую инициализацию (массивы, структуры).

#### realloc(void *ptr, size_t new_size)
Изменяет размер ранее выделенного блока.
* если рядом есть свободное место — просто расширяет блок на месте
* если нет — выделяет новый блок нужного размера, копирует туда старые данные, освобождает старый блок
* если ptr == NULL — ведёт себя как malloc
* если new_size == 0 — ведёт себя как free
Возвращает указатель на новый блок (может отличаться от старого!) или NULL при ошибке.

#### free(void *ptr)
Возвращает ранее выделенный блок обратно в кучу.
После вызова указатель становится недействительным (dangling pointer).
Повторный free одного и того же блока — неопределённое поведение.
free(NULL) — безопасно, ничего не делает.

### Оператор sizeof — теория
Это оператор (не функция!) времени компиляции.
Возвращает количество байтов, которое занимает объект или тип (результат имеет тип size_t).

Работает с типами: sizeof(int), sizeof(double), sizeof(struct Node)

Работает с выражениями: sizeof *ptr — размер объекта, на который указывает ptr

Не вычисляет выражение: sizeof(a++) — значение a не изменится

Для массива, объявленного в текущей области видимости, sizeof(arr) даёт полный размер массива в байтах

### Структура (struct)
Структура — это пользовательский тип данных, который объединяет несколько переменных (членов) разных типов под одним именем.
Члены хранятся в памяти последовательно (с возможным выравниванием — padding).

#### Определение:
Объявление типа: struct Tag { int a; double b; char c; };
Создание объекта: struct Tag obj;
Анонимная структура (редко): struct { ... } obj;

Размер структуры
sizeof(struct Tag) ≥ сумма размеров членов.

Компилятор добавляет padding для выравнивания членов по границам, соответствующим их типу (обычно по 4 или 8 байтам на x64).

Последний член может иметь padding до размера самого большого члена (для массивов структур).

Чтобы минимизировать padding — располагать члены от большего типа к меньшему.

#### Особенности использования

* Доступ к члену: оператор «.» (точка) — obj.member
* Можно копировать структуры присваиванием: struct1 = struct2; (побитовое копирование)
* Массив структур хранится плотно (с учётом padding одного элемента)
* Гибкий член массива (flexible array member, с C99): последний член char data[]; — размер структуры не включает его, выделяется динамически.

### Объединение (union)
Объединение — тип данных, где все члены занимают одну и ту же область памяти (начиная с одного адреса).

Размер union равен размеру самого большого члена (плюс возможный padding до выравнивания).

Смысл: экономия памяти или интерпретация одних и тех же байтов разными типами (например, типизированный вариант void*).

Определение аналогично struct, но ключевое слово union.

Доступ к члену тоже через «.»

В любой момент времени «активен» только последний записанный член — чтение другого даёт неопределённое поведение (но на практике работает для low-level).

Операций добавления/удаления элемента, доступа к элементу по индексу и по значению, их вычислительные сложности. Реализация взаимодействия с динамическим массивом на языке C

#### Динамический массив — это структура данных, имитирующая массив переменного размера.
В C нет встроенных векторов (как std::vector в C++), поэтому реализуется вручную через указатель и динамическую память.

#### Основные компоненты структуры
Обычно оформляется как структура:
* указатель на элементы (типа T*)
* текущий размер (size_t size) — сколько элементов хранится
* ёмкость (size_t capacity) — сколько памяти выделено (capacity ≥ size)

### Создание
Выделяем начальную память (например, на 4–8 элементов):
````C
typedef struct {
    int *data;
    size_t size;
    size_t capacity;
} DynArray;

DynArray* create_array(size_t initial_capacity) {
    DynArray *arr = malloc(sizeof(DynArray));
    arr->data = malloc(initial_capacity * sizeof(int));
    arr->size = 0;
    arr->capacity = initial_capacity;
    return arr;
}
````

#### Перераспределение памяти 
Когда size == capacity, нужно увеличить ёмкость. Для этого используется
функция realloc.
#### realloc:
● Изменяет размер ранее выделенного блока памяти.

● Принимает указатель на старый блок памяти и новый размер в байтах.

● Возвращает указатель на новый блок памяти (возможно, перемещенный в
другое место) или NULL в случае ошибки.

● Содержимое старого блока памяти до нового размера копируется в новый блок

* Стандартная стратегия — умножение на коэффициент роста (обычно 1.5 или 2).
* Сложность: амортизированная O(1) при добавлении, худшая O(n) при одном перераспределении.
### Основные операции и их вычислительная сложность
* Доступ по индексу
``arr->data[index]`` — O(1)

Проверка границ обычно на пользователе (или вручную).

#### Добавление элемента в конец (push_back) 

* * Если size < capacity — просто arr->data[size++] = value; O(1)

* * Если size == capacity — realloc на новую ёмкость, копирование старых данных — O(n)
* Амортизированная сложность при последовательном добавлении n элементов: O(1) на операцию.

#### Удаление элемента из конца (pop_back) - O(1)
````
arr->size--; 
````

Опционально уменьшать capacity (редко, чтобы избежать частых realloc).

#### Удаление элемента по индексу
Сдвиг всех последующих элементов влево — O(n)
(аналогично memmove).

#### Поиск элемента по значению
Линейный поиск — O(n) в худшем и среднем случае.

<hr>

#### Полный пример реализации push_back с ростом ×2
````C
void push_back(DynArray *arr, int value) {
    if (arr->size == arr->capacity) {
        size_t new_cap = arr->capacity ? arr->capacity * 2 : 8;
        int *new_data = realloc(arr->data, new_cap * sizeof(int));
        if (!new_data) { /* обработка ошибки */ return; }
        arr->data = new_data;
        arr->capacity = new_cap;
    }
    arr->data[arr->size++] = value;
}
````

#### Освобождение памяти
````C
void destroy_array(DynArray *arr) {
    free(arr->data);
    free(arr);
}
````

Списочные структуры данных представляют собой последовательность элементов, где
каждый элемент хранит данные и указатель (или указатели) на следующий (и предыдущий в случае двусвязного списка) элемент.


Односвязный список (singly linked list) — это структура данных, состоящая из узлов, где
каждый узел содержит данные и указатель на следующий узел в списке. Последний
узел списка указывает на NULL.

Структура узла:
````C
typedef struct Node {
    int data;
    struct Node *next;
} Node;
````

Операции и сложности:

* Доступ по индексу — O(n) (проходим по цепочке)
* Поиск по значению — O(n) в худшем случае
* Вставка в начало — O(1)
* Вставка в конец — O(n) (если нет указателя на хвост) или O(1) (если есть)
* Вставка после заданного узла — O(1)
* Удаление первого элемента — O(1)
* Удаление по значению/после узла — O(n) поиск + O(1) удаление

### Двусвязный список (doubly linked list)
Каждый узел содержит данные, указатель на следующий и на предыдущий узел.
Структура узла:
````C
typedef struct DNode {
    int data;
    struct DNode *next;
    struct DNode *prev;
} DNode;
````
Операции и сложности (по сравнению с односвязным):

* Доступ по индексу — всё ещё O(n)
* Поиск по значению — O(n)
* Вставка/удаление перед/после заданного узла — O(1) (не нужно искать предыдущий)
* Удаление последнего элемента — O(1) (если есть указатель на хвост)

Можно проходить в обе стороны

### Кольцевой список (circular linked list)

Последний узел указывает на первый. Бывает односвязным или двусвязным.

#### Особенности:
* Нет явного конца → цикл for(;;) с проверкой возврата к head
* Вставка/удаление в конец — O(1) при наличии указателя на хвост

Часто используется в планировщиках задач (round-robin)

#### Сложности операций (общие для всех типов списков):
* Доступ по индексу — O(n)
* Поиск по значению — O(n)
* Вставка/удаление в начало — O(1)
* Вставка/удаление в конец — O(1) с указателем на хвост, иначе O(n)
* Вставка/удаление после заданного узла — O(1)

### Реализация базовых операций (односвязный список)
Вставка в начало:
````C
void push_front(Node **head, int value) {
    Node *new_node = malloc(sizeof(Node));
    new_node->data = value;
    new_node->next = *head;
    *head = new_node;
}
````

Удаление первого элемента:
````C
void pop_front(Node **head) {
    if (*head == NULL) return;
    Node *temp = *head;
    *head = (*head)->next;
    free(temp);
}
````

Удаление после заданного узла:
````C
void delete_after(Node *prev) {
    if (prev == NULL || prev->next == NULL) return;
    Node *temp = prev->next;
    prev->next = temp->next;
    free(temp);
}
````

## Списочные структуры данных. Очереди и стеки. Организация очереди и стека на основе односвязного списка. Вычислительные сложности операций доступа к элементу, вставки и удаления элементов для стека и очереди, их реализация на языке C. Алгоритм проверки баланса скобок в выражении.

Очереди и стеки — это абстрактные типы данных (АТД), которые представляют собой коллекции элементов, доступ к которым осуществляется по определенным правилам.

### Очереди
Очередь (queue) — это структура данных, работающая по принципу "первым пришел
— первым ушел" (FIFO - First In, First Out). Элементы добавляются в конец очереди
(enqueue) и удаляются из начала (dequeue)

* enqueue(element): Добавляет элемент в конец очереди.
* dequeue(): Удаляет и возвращает элемент из начала очереди.
* front(): Возвращает элемент из начала очереди без удаления.
* isEmpty(): Проверяет, пуста ли очередь.
* size(): Возвращает количество элементов в очереди.

````C
typedef struct {
    Node *head;
    Node *tail;
} Queue;

void queue_enqueue(Queue *q, int value) {
    Node *n = malloc(sizeof(Node));
    n->data = value;
    n->next = NULL;
    if (q->tail == NULL) {
        q->head = q->tail = n;
    } else {
        q->tail->next = n;
        q->tail = n;
    }
}

int queue_dequeue(Queue *q) {
    Node *temp = q->head;
    int value = temp->data;
    q->head = q->head->next;
    if (q->head == NULL) q->tail = NULL;
    free(temp);
    return value;
}
````


### Стек
Стек (stack) — это структура данных, работающая по принципу "последним пришел —
первым ушел" (LIFO - Last In, First Out). Элементы добавляются и удаляются с
вершины стека.

Операции:
* push(element): Добавляет элемент на вершину стека.
* pop(): Удаляет и возвращает элемент с вершины стека.
* peek(): Возвращает элемент с вершины стека без удаления.
* isEmpty(): Проверяет, пуст ли стек.
* size(): Возвращает количество элементов в стеке

````C
typedef struct {
    Node *top;
} Stack;

void stack_push(Stack *s, int value) {
    Node *n = malloc(sizeof(Node));
    n->data = value;
    n->next = s->top;
    s->top = n;
}

int stack_pop(Stack *s) {
    Node *temp = s->top;
    int value = temp->data;
    s->top = temp->next;
    free(temp);
    return value;
}
````

Для стека, добавляем в структуру: 
````C

    Node *top;

````

````C

````

### Инфиксная форма (infix)
Операторы располагаются между операндами (например, ``2 + 3 × 4``).
Это привычная нам математическая запись.

Недостатки: нужны скобки для задания порядка вычислений, приоритет операций должен учитываться вручную.
### Постфиксная форма (postfix, обратная польская нотация, RPN)
* Операторы стоят после своих операндов (например, 2 3 4 × +).
#### Преимущества:
* скобки не нужны вообще
* однозначный порядок вычислений
* идеально подходит для вычисления на стеке

Пример: ``(2 + 3) × 4 → 2 3 + 4 ×``

### Префиксная форма (prefix, польская нотация, PN)
Операторы стоят перед своими операндами (например, ``+ 2 × 3 4``).

Преимущества те же, что у постфиксной: нет скобок, однозначность.

! Используется реже, чем постфиксная, но применяется в некоторых компиляторах и Lisp-подобных языках.

### Алгоритм преобразования из инфиксной в постфиксную форму (Shunting Yard)
Используем стек для операторов и выходную очередь.
#### Правила:
* Число → сразу в выход
* '(' → в стек
* ')' → выталкиваем из стека в выход всё до '('
* Оператор → пока на вершине стека оператор с приоритетом ≥ текущего — выталкиваем в выход, затем кладём текущий
* В конце — выталкиваем остаток стека в выход

Сложность: O(n)

### Алгоритм вычисления постфиксного выражения
Используем один стек для операндов.

Идём слева направо:
* Число → кладём в стек
* Оператор → берём два верхних числа (правое — второй операнд), выполняем операцию, результат обратно в стек
* В конце в стеке остаётся единственное число — результат.

* Сложность: O(n)

Дерево – конечное множество объектов Т, состоящее из одного или более узлов, для которых выполняются следующие условия:
* имеется один специально выделенный узел, называемый корнем данного дерева;
* остальные узлы (исключая корень) содержатся в m попарно непересекающихся множествах T1, …,Tm, каждое из которых в свою очередь является деревом. Деревья T1, ...,Тm называются поддеревьями данного корня.


Дерево общего вида (general tree), также известное как n-арное дерево, — это
древовидная структура данных, в которой каждый узел может иметь любое количество
дочерних узлов. 

В отличие от бинарного дерева, где каждый узел имеет не более двух
потомков, в общем дереве количество потомков не ограничено

### Основные операции над деревом общего вида
* Создание узла, O(1)
* Добавление потомка к узлу, O(1) - если есть список потомков как связный список | O(n)
* Обход дерева (прямой, обратный, по уровням), O(n)
* ``Обход в глубину (Depth-First Traversal - DFT): Предусматривает
посещение узлов как можно "глубже" в поддереве, прежде чем
переходить к другим ветвям. Существуют разные варианты DFT, такие
как preorder (сначала посещается корень, затем рекурсивно обходятся
поддеревьяю. К-Л-П), inorder (в бинарных деревьях, ЛКП) и postorder (сначала
рекурсивно обходятся поддеревья, затем посещается корень, ЛПК).
Обход в ширину (Breadth-First Traversal - BFT): Предусматривает
посещение узлов уровень за уровнем, начиная с корня. Обычно
реализуется с использованием очереди.``


* Поиск узла по значению, O(n)
* Удаление поддерева, O(n) в худшем случае
* Подсчёт количества узлов, высоты, листьев и т.д., O(n)

## Отношение порядка на пространстве элементов. Кастомизация операции отношения порядка для заданного контекста в языке C. Быстрая сортировка и функция qsort. Алгоритм дихотомического поиска, его алгоритмическая сложность.

Отношение порядка — это правило, по которому элементы можно упорядочить.
Для сортировки важно понимать, что сравнение элементов должно быть:

* Рефлексивным (a ≤ a) — в программировании обычно не используется явно.
* Антисимметричным (если a ≤ b и b ≤ a → a = b)
* Транзитивным (если a ≤ b и b ≤ c → a ≤ c)
* Полным (любые два элемента сравнимы — важно для корректной сортировки)

В программировании это сводится к функции сравнения, которая:
1. принимает два элемента
2. решает, какой «меньше»
3. возвращает отрицательное / ноль / положительное значение

То есть в C отношение порядка задаётся функцией сравнения (компаратором).

### Кастомизация операции отношения порядка в C
В C нет встроенных средств для перегрузки операторов, поэтому "кастомизация порядка" делается с помощью функций, которые мы передаём как указатели.

``int cmp(const void *a, const void *b);`` возвращает < 0 — если a < b; 0 — если a == b, > 0 - если a > b

Примеры:
````C
int cmp_int(const void *a, const void *b) {
    return *(const int*)a - *(const int*)b;
}

int cmp_str(const void *a, const void *b) {
    return strcmp(*(const char* const*)a, *(const char* const*)b);
}

typedef struct { int x, y; } Point;
int cmp_point_by_x(const void *a, const void *b) {
    const Point *p1 = a, *p2 = b;
    return (p1->x > p2->x) - (p1->x < p2->x);  // безопасно от переполнения
}
````

* base — адрес массива

Алгоритм внутри — быстрая сортировка (quicksort):

Применяется только к отсортированным массивам.


### Алгоритм дихотомического (бинарного) поиска
Идея:
1. Найти середину.
2. Сравнить искомое значение со значением середины.
3. Если меньше — искать в левой половине.
4. Если больше — в правой.
5. Повторять пока не останется один элемент или не найдём искомое.

#### Сложность
Время: O(log n)
Память: O(1) (итеративная версия)

Прототип в Си:
````C

              size_t nmemb, size_t size,
              int (*compar)(const void *, const void *));
````

Быстрое возведение в степень (pow для целых чисел)
Стандартной функции pow из <math.h> для целых лучше не пользоваться — она работает с double и медленная.
На практике используют алгоритм «бинарное возведение в степень» (Exponentiation by squaring).
Алгоритм (итеративный вариант):

````
long long binpow_iter(long long a, long long n) {
    long long res = 1;
    while (n > 0) {
        if (n & 1)          // если младший бит = 1, чётн
            res *= a;
        a *= a;             // переходим к следующему биту
        n >>= 1;    // отсекаем ласт бит
    }
    return res;
}
````

Итого: O(log b)

### Обход дерева: Существуют различные способы обхода дерева, включая:
○ Обход в глубину (Depth-First Traversal - DFT): Предусматривает
посещение узлов как можно "глубже" в поддереве, прежде чем
переходить к другим ветвям.

* inorderTraversal — обход узлов в отсортированном порядке, ЛКП
* preorderTraversal — обход узлов в порядке: вершина, левое поддерево, правое поддерево,
* postorderTraversal — обход узлов в порядке: левое поддерево, правое поддерево, вершина.

○ Обход в ширину (Breadth-First Traversal - BFT): Предусматривает

реализуется с использованием очереди.

### Иные операции

#### Восстановление свойств кучи
В случае, если свойства кучи были нарушены для 1 элемента, мы можем восстановить их:
Поменяться местами с наибольшим потомком;

В случае если мы достигли нижней границы кучи (нет потомков), то останавливаемся;

Если есть потомки имеющие значение больше чем у нас, повторяем п. 2.

#### Поиск элемента
!В силу свойств бинарной кучи, поиск ее максимума занимает O(1), такой элемент находится всегда в корне


Начинаем с корня, сравниваем ключ и идём влево/вправо.
* Сложность: O(h), где h — высота дерева. Средний случай: O(log n), худший: O(n).
* Связь с дихотомическим поиском: в сбалансированном BST поиск эквивалентен бинарному поиску в отсортированном массиве.
#### Вставка элемента
Идём как при поиске до NULL и создаём новый узел.
Сложность: O(h) → средний O(log n), худший O(n).


#### Удаление элемента
* Нет детей — просто удаляем
* Один ребёнок — заменяем узел на ребёнка
* Два ребёнка — находим минимальный в правом поддереве (или максимальный в левом), копируем его ключ, удаляем этот минимальный узел
* Сложность: O(h) → средний O(log n), худший O(n).

!!
При удалении максимума мы извлекаем его из кучи и ставим на его место наименьший элемент (последний элемент в массиве)
Далее мы по описанному ранее алгоритму восстанавливаем свойства кучи для корня



#### Создание дерева 
— пустое дерево: root = NULL. O(1)

Наивным способом создания кучи является поочередное добавление всех ее элементов, но данный подход имеет сложность O(Nlog2(N)).
Более рациональный подход имеет сложность O(N) и заключается в том, что мы уже для имеющегося массива начинаем производить восстановление свойств кучи, начиная с предпоследнего слоя (так как последний слой по умолчанию удовлетворяет свойствам кучи) и до корня кучи (нулевого элемента массива).

#### Удаление всего дерева 
Рекурсивный post-order: освободить левое, правое, затем узел. Сложность O(n).

#### Слияние

Вариант с последовательным добавлением элементов II кучи в I имеет сложность O(mlog(n+m)).
Подход со сложностью O(n + m) заключается в добавлении в конец массива, содержащего I кучу, массива, содержащего II кучу и вызов процедуры восстановления свойств кучи, начиная с (n+m)/2 элемента, до корня кучи.


### Реализация основных операций на C

````C
Node* create_node(int key) {
    Node* node = malloc(sizeof(Node));
    node->key = key;
    node->left = node->right = NULL;
    return node;
}

Node* insert(Node* root, int key) {
    if (root == NULL) return create_node(key);
    if (key < root->key)
        root->left = insert(root->left, key);
    else if (key > root->key)
        root->right = insert(root->right, key);
    return root;
}

Node* find(Node* root, int key) {
    if (root == NULL || root->key == key) return root;
    if (key < root->key) return find(root->left, key);
    return find(root->right, key);
}

Node* find_min(Node* root) {
    while (root && root->left) root = root->left;
    return root;
}

Node* delete(Node* root, int key) {
    if (root == NULL) return root;
    if (key < root->key)
        root->left = delete(root->left, key);
    else if (key > root->key)
        root->right = delete(root->right, key);
    else {  // нашли
        if (root->left == NULL) {
            Node* temp = root->right;
            free(root);
            return temp;
        } else if (root->right == NULL) {
            Node* temp = root->left;
            free(root);
            return temp;
        }
        // два ребёнка
        Node* temp = find_min(root->right);
        root->key = temp->key;
        root->right = delete(root->right, temp->key);
    }
    return root;
}

void inorder(Node* root) {      // отсортированный вывод
    if (root) {
        inorder(root->left);
        printf("%d ", root->key);
        inorder(root->right);
    }
}

void free_tree(Node* root) {
    if (root) {
        free_tree(root->left);
        free_tree(root->right);
        free(root);
    }
}

Декартово дерево (treap) – хранит пары (Х, Y) таким образом, что оно является BST по X и heap по Y


Каждый узел имеет два поля:

key — для свойства BST (левое < узел < правое)
priority — случайное число, для свойства min-heap (приоритет родителя ≥ приоритетов детей)

Случайные приоритеты обеспечивают ожидаемую высоту O(log n), делая все операции в среднем O(log n).

#include <stdlib.h>
#include <time.h>

typedef struct Node {
    int key;
    int priority;              // случайный приоритет
    struct Node *left;
    struct Node *right;
} Node;

// Генерация случайного приоритета 
Node* create_node(int key) {
    Node* node = malloc(sizeof(Node));
    node->key = key;
    node->priority = rand();   // чем больше диапазон, тем лучше
    node->left = node->right = NULL;
    return node;
}

### Основные операции и их сложности
Все операции — ожидаемая O(log n), худшая O(n).

### Создание

Создание: все элементы в кучу – по Y, а далее просто вставляем их в дерево по Х.

Отсортируем все n узлов по убыванию приоритета.

Сложность сортировки: O(n log n).

Пусть после сортировки первый узел (с максимальным приоритетом) — y_k.

Делаем y_k корнем всего дерева.

Рекурсивно строим левое и правое поддеревья:

* Все узлы с key < y_k.key → идут в левое поддерево
* Все узлы с key > y_k.key → идут в правое поддерево
* При этом порядок приоритетов в подмассивах уже отсортирован по убыванию (потому что мы взяли подпоследовательность из отсортированного списка).


#### Поиск элемента
Обычный BST-поиск по key.
* Связь с дихотомическим поиском: идентичен бинарному поиску в отсортированном массиве при сбалансированном дереве.
* Сложность: ожидаемая O(log n).
#### Вставка элемента
Вставляем как в обычное BST по key (до листа).

Выполняем повороты, чтобы восстановить свойство кучи по priority.

Повороты:
* left_rotate (если правый ребёнок имеет больший приоритет)
* right_rotate (если левый ребёнок имеет больший приоритет)

#### Удаление элемента
1. Находим узел по key.
2. Поворачиваем его вниз, пока он не станет листом (нарушаем кучу, чтобы дети имели больший приоритет).
3. Удаляем лист.

Удаление всего дерева — рекурсивный post-order, O(log n) до O(n) в худшем случае

### Split
1. Рассмотрим случай, в котором требуется разрезать дерево по ключу, большему ключа корня. Посмотрим, как будут устроены результирующие деревья  T1 и T2:
2. T1 : левое поддерево T1 совпадёт с левым поддеревом T. Для нахождения правого поддерева T1, нужно разрезать правое поддерево T на TR1 и TR2 по ключу k и взять  TR1
3. T2 совпадёт с TR2. 
* Случай, в котором требуется разрезать дерево по ключу, меньше либо равному ключа в корне, рассматривается симметрично.

### Merge
Слить два декартовых дерева в одно. 

Все ключи в первом(левом) дереве должны быть меньше, чем ключи во втором(правом). В результате получается дерево, в котором есть все ключи из первого и второго деревьев:  ``merge(T1,T2)→{T}``.

Пусть нужно слить деревья T1 и T2. Тогда, очевидно, у результирующего дерева T есть корень. Корнем станет вершина из T1 или T2 с наибольшим приоритетом y. Но вершина с самым большим y из всех вершин деревьев T1 и T2 может быть только либо корнем  T1, либо корнем T2.
1. Рассмотрим случай, в котором корень T1 имеет больший y, чем корень T2. Случай, в котором корень T2 имеет больший y, чем корень T1, симметричен этому.
2. Если y корня T1 больше y корня T2, то он и будет являться корнем. Тогда левое поддерево T совпадёт с левым поддеревом T1 . Справа же нужно подвесить объединение правого поддерева T1 и дерева T2.

Операции вставки элемента, поиска минимума/максимума, удаления минимума/максимума, слияния с разрушением и без разрушения исходных бинарных приоритетных очередей; их вычислительные сложности и реализация на языке C

Приоритетная очередь – это абстрактная структура данных, которая поддерживает
операции добавления элемента и извлечения элемента с наивысшим (или
наименьшим) приоритетом. В отличие от обычной очереди (FIFO) или стека (LIFO),
приоритетная очередь упорядочивает элементы на основе их приоритета, а не
времени добавления.

### Бинарная куча
Бинарная куча (Binary Heap) – это один из способов реализации
приоритетной очереди, использующий структуру данных в виде бинарного дерева.

* Значение в любой вершине не меньше (относительно введенного компаратора), чем значения её потомков. (ДЛЯ MIN-HEAP: НЕ БОЛЬШЕ ЗНАЧ. ПОТОМКОВ)
* На i-ом слое 2i вершин, кроме последнего. Слои нумеруются с нуля.
* Последний слой заполняется слева направо (как показано на рисунке)

Чаще всего бинарную кучу реализуют на динамическом массиве
Хранение на базе массива:
* Left(i) – 2*i + 1
* Right(i) - 2*i+2

````C
typedef struct {
    int *data;      // динамический массив элементов
    size_t size;    // текущее количество элементов
    size_t capacity;// выделенная память
} BinaryHeap;
````

#### Вставка O(log(n))

1. Добавление элемента в конец массива;
2. Сравнение с родителем;
3. Если родитель больше или мы дошли до корня, то вставка окончена;
4. В случае если родитель меньше, то меняемся местами с родителем и возвращаемся к п.2.

````C
void heap_push(MaxHeap *h, int val) {
    if (h->size == h->capacity) {
        h->capacity = h->capacity ? h->capacity * 2 : 4; // нач. емкость 4
        h->data = realloc(h->data, h->capacity * sizeof(int));
    }
    size_t i = h->size++;
    h->data[i] = val;
    while (i > 0 && h->data[i] > h->data[parent(i)]) {
        swap(&h->data[i], &h->data[parent(i)]);
        i = parent(i);
    }
}
````

вспомогательные:
````C
static void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
}

static size_t parent(size_t i) { return (i - 1) / 2; }
static size_t left(size_t i)   { return 2 * i + 1; }
static size_t right(size_t i)  { return 2 * i + 2; }
````

#### Поиск максимума
В силу свойств бинарной кучи, поиск ее максимума занимает O(1), так как такой элемент находится всегда в её корне

``int heap_max(const MaxHeap *h) {
    return h->data[0];   
}``

#### Восстановление свойств кучи
если свойства кучи были нарушены для 1 элемента:
1. Поменяться местами с наибольшим потомком;
2. В случае если мы достигли нижней границы кучи (нет потомков), то останавливаемся;
3. Если есть потомки имеющие значение больше чем у нас, повторяем п. 2.

#### Удаление максимума
При удалении максимума мы извлекаем его из кучи и ставим на его место наименьший элемент (последний элемент в массиве)
Далее мы по описанному ранее алгоритму восстанавливаем свойства кучи для корня

O(log n)

````C
int heap_pop(MaxHeap *h) {
    int max_val = h->data[0];
    h->data[0] = h->data[--h->size];
    size_t i = 0;
    while (left(i) < h->size) {
        size_t largest = i;
        size_t l = left(i), r = right(i);
        if (l < h->size && h->data[l] > h->data[largest])
            largest = l;
        if (r < h->size && h->data[r] > h->data[largest])
            largest = r;
        if (largest == i) break;
        swap(&h->data[i], &h->data[largest]);
        i = largest;
    }
    return max_val;
}
````


#### Удаление произвольного элемента O(n)

В классической бинарной приоритетной очереди удаление произвольного элемента не является стандартной операцией, потому что куча не поддерживает быстрый поиск по значению (это не BST).

Аналогично: мы замещаем удаленный элемент минимальным возможным (конец массива) и восстанавливаем для него свойства кучи по описанному ранее алгоритму

#### Слияние куч

##### С разрушением исходных (destructive merge)
* Самый эффективный способ — использовать heap_push последовательно: O((n+m) log(n+m))
* Или построить кучу из объединённого массива за O(n+m) (heapify).

##### Без разрушения исходных (non-destructive merge) 
Создаём новую кучу и вставляем элементы из обеих — O((n+m) log(n+m))


````C
MaxHeap* heap_merge_non_destructive(const MaxHeap *h1, const MaxHeap *h2) {
    MaxHeap *result = malloc(sizeof(MaxHeap));
    result->capacity = h1->size + h2->size;
    result->data = malloc(result->capacity * sizeof(int));
    result->size = 0;
    for (size_t i = 0; i < h1->size; i++)
        heap_push(result, h1->data[i]);
    for (size_t i = 0; i < h2->size; i++)
        heap_push(result, h2->data[i]);
    return result;
}
````

````C
void heap_merge_destructive(MaxHeap *dest, MaxHeap *src) {
    if (dest->size + src->size > dest->capacity) {
        dest->capacity = dest->size + src->size;
        dest->data = realloc(dest->data, dest->capacity * sizeof(int));
    }
    memcpy(dest->data + dest->size, src->data, src->size * sizeof(int));
    dest->size += src->size;
    for (size_t i = dest->size / 2; i != (size_t)-1; i--) {
        size_t j = i;
        while (left(j) < dest->size) {
            size_t largest = j;
            size_t l = left(j), r = right(j);
            if (l < dest->size && dest->data[l] > dest->data[largest])
                largest = l;
            if (r < dest->size && dest->data[r] > dest->data[largest])
                largest = r;
            if (largest == j) break;
            swap(&dest->data[j], &dest->data[largest]);
            j = largest;
        }
    }
}
````## Приоритетные очереди. Левосторонняя приоритетная очередь, её реализация на базе структуры данных вида бинарное дерево. Операции вставки элемента, поиска минимума/максимума, удаления минимума/максимума, слияния с разрушением и без разрушения исходных левосторонних приоритетных очередей; их вычислительные сложности и реализация на языке C.

Левосторонняя куча — это разновидность приоритетной очереди, реализованная в виде бинарного дерева (обычно min-heap), в котором, помимо свойства кучи, выполняется дополнительное левостороннее свойство:
* ! У каждого узла расстояние до ближайшего листа в левом поддереве (npl — null path length) не меньше, чем в правом.
Это гарантирует, что правое поддерево всегда «легче», и слияние работает эффективно.

### Слияние 

* Данная операция является ключевой в схеме работы левостороннего дерева и через нее выражаются все остальные операции.
* Во время слияния двух левосторонних деревьев мы должны поддерживать их основное свойство
* сравниваем корни обоих деревьев, меньший становится новым корнем, а большее дерево сливается с правым поддеревом меньшего 


* Алгоритм:
1. Ориентируясь на то, что дерево левостороннее рекурсивно начинаем слияние нового дерева с правым поддеревом;
2. Если “левостороннесть” кучи была нарушена, меняем получившегося правого и левого ребенка местами;
3. Пересчитываем расстояние до ближайшей свободной позиции

### Вставка
1. Обозначаем новый элемент как левостороннее дерево с 1 вершиной;
2. Производим слияние имеющегося дерева с вновь созданным.

* Сложность: O(log(n))

### Извлечение минимального элемента
1. Извлекаем текущий корень дерева;
2. Производим слияние левого и правого поддеревьев.

* Сложность: O(log(n))

### Удаление элемента
Алгоритм:

1. Найти узел (O(log n) в худшем)
2. Удалить его как минимум (повернуть вверх по приоритету или использовать другой трюк)

! удаление произвольного элемента не считается стандартной операцией leftist heap (в отличие от Fibonacci heap).

* Сложность: O(log(n))

### Построение кучи
Левосторонняя куча строится последовательным добавлением в нее всех необходимых элементов;

* Сложность: O(n) аморт. (через попарные слияния)

````C
#include <stdio.h>
#include <stdlib.h>

// Узел левосторонней кучи (min-heap)
typedef struct Node {
    int key;                   // значение (приоритет)
    int npl;                   // null path length
    struct Node *left;
    struct Node *right;
} Node;

// Представление приоритетной очереди — указатель на корень
typedef Node* LeftistHeap;

// Вспомогательная функция: npl(NULL) = -1
static int get_npl(const Node *t) {
    return t ? t->npl : -1;
}

// Ключевая операция: слияние двух левосторонних куч (с разрушением исходных)
static Node* merge(LeftistHeap h1, LeftistHeap h2) {
    if (!h1) return h2;
    if (!h2) return h1;

    // Обеспечиваем, что h1->key ≤ h2->key
    if (h1->key > h2->key) {
        Node *temp = h1;
        h1 = h2;
        h2 = temp;
    }

    // Рекурсивно сливаем h2 с правым поддеревом h1
    h1->right = merge(h1->right, h2);

    // Восстанавливаем левостороннее свойство
    if (get_npl(h1->left) < get_npl(h1->right)) {
        Node *temp = h1->left;
        h1->left = h1->right;
        h1->right = temp;
    }

    // Пересчитываем npl
    h1->npl = get_npl(h1->right) + 1;

    return h1;
}

// Поиск минимума — O(1)
int find_min(LeftistHeap h) {
    if (!h) {
        fprintf(stderr, "Heap is empty\n");
        exit(1);
    }
    return h->key;
}

// Вставка элемента — O(log n)
LeftistHeap insert(LeftistHeap h, int key) {
    Node *new_node = malloc(sizeof(Node));
    new_node->key = key;
    new_node->npl = 0;
    new_node->left = new_node->right = NULL;
    return merge(h, new_node);
}

// Удаление минимума — O(log n)
int delete_min(LeftistHeap *h) {
    if (!*h) {
        fprintf(stderr, "Heap is empty\n");
        exit(1);
    }
    int min_val = (*h)->key;
    Node *left = (*h)->left;
    Node *right = (*h)->right;
    free(*h);
    *h = merge(left, right);
    return min_val;
}

// Освобождение всей кучи (post-order)
void free_heap(LeftistHeap h) {
    if (h) {
        free_heap(h->left);
        free_heap(h->right);
        free(h);
    }
}


// копирование не разрушая
// Вспомогательная функция — копирование узла
static Node* copy_node(const Node *src) {
    if (!src) return NULL;
    Node *new = malloc(sizeof(Node));
    new->key = src->key;
    new->npl = src->npl;
    new->left = src->left;    // сначала ссылаемся на исходные поддеревья
    new->right = src->right;
    return new;
}

Node* merge_non_destructive(Node *h1, Node *h2) {
    if (!h1) return copy_node(h2);  // если один пуст — копируем другой
    if (!h2) return copy_node(h1);
    Node *root;
    if (h1->key <= h2->key) {
        root = copy_node(h1);
        root->right = merge_non_destructive(h1->right, h2);
    } else {
        root = copy_node(h2);
        root->right = merge_non_destructive(h1, h2->right);
    }
    // Восстанавливаем левостороннее свойство
    if (get_npl(root->left) < get_npl(root->right)) {
        Node *temp = root->left;
        root->left = root->right;
        root->right = temp;
    }
    // Пересчитываем npl
    root->npl = get_npl(root->right) + 1;
    return root;
}
````## Приоритетные очереди. Косая приоритетная очередь, её реализация на базе структуры данных вида бинарное дерево. Операции вставки элемента, поиска минимума/максимума, удаления минимума/максимума, слияния с разрушением и без разрушения исходных косых приоритетных очередей; их вычислительные сложности и реализация на языке C.

Косая куча — это саморегулирующаяся бинарная куча (обычно min-heap), в которой слияние является ключевой операцией.

В отличие от leftist heap, здесь нет дополнительного поля npl.

Саморегуляция достигается за счёт безусловной перестановки левого и правого ребёнка при каждом слиянии.

Это даёт амортизированную сложность O(log n) для всех операций (не гарантированную, как в leftist, но очень хорошую на практике).

•Обладает такими же свойствами, как и левосторонняя куча, но имеет отличие в процессе слияния двух куч.

````C
typedef struct Node {
    int key;                   // значение (приоритет)
    struct Node *left;
    struct Node *right;
} Node;

typedef Node* SkewHeap;        // очередь — указатель на корень
````

Операции и вычислительные сложности

* Поиск минимума/максимума — O(1) (корень кучи)
* Вставка элемента — O(log n) амортизированная (создать узел + слияние)
* Удаление минимума/максимума — O(log n) амортизированная (удалить корень + слияние поддеревьев)
* Слияние двух куч — O(log n) амортизированная. Схоже с алгоритмом для левосторонней кучи, но перестановка левого и правого поддерева БЕЗУСЛОВНА
  
* (
1. Если одна из куч пуста — возвращаем другую.
2. Сравниваем корни двух куч. Если корень второй кучи меньше — меняем кучи местами (чтобы корень результата имел минимальный ключ).
3. Корнем результата становится корень первой кучи.
4. Безусловно меняем местами левое и правое поддеревья этого корня (это ключевое действие).
5. Рекурсивно сливаем вторую кучу с тем поддеревом, которое теперь стало правым (после перестановки).
6. Присваиваем результат рекурсии правому ребёнку корня.
7. Возвращаем новый корень.
  )

Реализация на Си

````C
#include <stdlib.h>

// Ключевая операция: слияние с разрушением (амортизированная O(log n))
SkewHeap merge(SkewHeap h1, SkewHeap h2) {
    if (!h1) return h2;
    if (!h2) return h1;

    // Обеспечиваем h1->key ≤ h2->key
    if (h1->key > h2->key) {
        SkewHeap temp = h1;
        h1 = h2;
        h2 = temp;
    }

    // Безусловная перестановка детей
    SkewHeap temp = h1->left;
    h1->left = h1->right;
    h1->right = temp;

    // Рекурсивно сливаем бывшее правое поддерево с h2
    h1->right = merge(h1->right, h2);

    return h1;
}

// Поиск минимума (min-heap)
int find_min(SkewHeap h) {
    if (!h) {
        // ошибка: пустая куча
        exit(1);
    }
    return h->key;
}

// Вставка
SkewHeap insert(SkewHeap h, int key) {
    SkewHeap new_node = malloc(sizeof(Node));
    new_node->key = key;
    new_node->left = new_node->right = NULL;
    return merge(h, new_node);
}

// Удаление минимума
int delete_min(SkewHeap *h) {
    if (!*h) {
        // ошибка
        exit(1);
    }
    int min_val = (*h)->key;
    SkewHeap left = (*h)->left;
    SkewHeap right = (*h)->right;
    free(*h);
    *h = merge(left, right);
    return min_val;
}

// Слияние без разрушения исходных (копируем путь слияния)
SkewHeap merge_non_destructive(SkewHeap h1, SkewHeap h2) {
    if (!h1) return h2 ? h2 : NULL;  // простая копия, если один пуст
    if (!h2) return h1;

    SkewHeap root;
    if (h1->key <= h2->key) {
        root = malloc(sizeof(Node));
        root->key = h1->key;
        root->left = h1->left;
        root->right = merge_non_destructive(h1->right, h2);
    } else {
        root = malloc(sizeof(Node));
        root->key = h2->key;
        root->left = h2->left;
        root->right = merge_non_destructive(h1, h2->right);
    }

    // Безусловная перестановка
    SkewHeap temp = root->left;
    root->left = root->right;
    root->right = temp;

    return root;
}

// Освобождение кучи
void free_skew_heap(SkewHeap h) {
    if (h) {
        free_skew_heap(h->left);
        free_skew_heap(h->right);
        free(h);
    }
}

### Биномиальное дерево
Биномиальное дерево ранга k — это дерево, состоящее из корня и k поддеревьев рангов 0, 1, …, k−1.

Число элементов в биномиальном дереве ранга k равно точно 2^k.

Ранг — это количество детей корня (и одновременно логарифм числа узлов).
### Биномиальная куча (binomial heap)
Это лес из биномиальных деревьев разных рангов с разными корневыми ключами, удовлетворяющих свойству кучи (min-heap или max-heap).

Корни деревьев связаны в кольцевой односвязный список (обычно по возрастанию ранга).
### Фибоначчиева куча (Fibonacci heap)
Это тоже лес деревьев общего вида, удовлетворяющих свойству кучи. Деревья связаны в кольцевой двусвязный список корней.

Дополнительные свойства:
* степень узла ограничена O(log n)
* пометка удалённых детей для отложенной очистки

### Операции и их вычислительные сложности:

* Поиск минимума — O(1) для обеих (указатель на минимальный корень или проход по списку корней)
* Вставка — O(log n) для биномиальной, амортизированная O(1) для фибоначчиевой
* Удаление минимума — O(log n) для биномиальной, амортизированная O(log n) для фибоначчиевой
* Слияние с разрушением — O(log n) для биномиальной, амортизированная O(1) для фибоначчиевой
* Слияние без разрушения — O(log n) для обеих (копирование списка корней или пути слияния)

Отношение эквивалентности на множестве элементов – это бинарное отношение,
которое удовлетворяет следующим свойствам:
* Рефлексивность: a ~ a (каждый элемент эквивалентен самому себе)
* Симметричность: Если a ~ b, то b ~ a
* Транзитивность: Если a ~ b и b ~ c, то a ~ c

Где ~ обозначает отношение эквивалентности. Отношение эквивалентности разбивает
множество на непересекающиеся классы эквивалентности. Все элементы внутри одного класса
эквивалентны друг другу, а элементы из разных классов – нет.

#### В C нет встроенной поддержки эквивалентности.

Для определения пользовательского отношения эквивалентности можно создать
функцию, которая принимает два указателя на элементы и возвращает true (1), если
они эквивалентны, и false (0) в противном случае

Например,
````C
int equiv_int(const void *a, const void *b) {
    return *(const int*)a == *(const int*)b;
}

int equiv_str(const void *a, const void *b) {
    return strcmp(*(const char* const*)a, *(const char* const*)b) == 0;
}
````

### Алгоритм дихотомического (бинарного) поиска

Дихотомический (бинарный) поиск – это алгоритм для поиска заданного значения в
отсортированном массиве. Он работает по принципу "разделяй и властвуй":
1. Сравниваем искомое значение со средним элементом массива.
2. Если значения равны, возвращаем индекс среднего элемента.3. Если искомое значение меньше среднего элемента, рекурсивно выполняем
   поиск в левой половине массива.
4. Если искомое значение больше среднего элемента, рекурсивно выполняем
   поиск в правой половине массива.

Алгоритмическая сложность: O(log n), где n - количество элементов в массиве. На каждом шаге размер области поиска уменьшается вдвое.

Существует и итеративная версия, реализуемая с циклом while/for. Ее сложность O(1)

Дерево скобочного выражения (или дерево арифметического выражения) – это
бинарное дерево, которое представляет арифметическое выражение. Оно позволяет
эффективно вычислять выражения и анализировать их структуру.
* Листья: Операнды (числа, переменные).
* Внутренние узлы: Операторы (+, -, *, /, ^ и т.д.).

### Алгоритм конвертации инфиксной формы в дерево

Существует несколько способов построения дерева из инфиксного выражения. Один
из распространенных подходов – использование алгоритма шунтирующего двора
(Shunting Yard Algorithm) для преобразования инфиксной формы в постфиксную
(обратную польскую запись), а затем построение дерева из постфиксной формы.

Также можно использовать рекурсивный спуск. Рассмотрим второй вариант, как более
наглядный для построения дерева.

#### Рекурсивный спуск:
1. Базовый случай: Если выражение состоит из одного операнда, создать лист с
   этим операндом и вернуть его.
2. Рекурсивный шаг:
   * Найти оператор с наименьшим приоритетом, который не находится
   внутри скобок. Этот оператор станет корнем поддерева.
   * Рекурсивно построить левое поддерево из части выражения слева от
   оператора.
   * Рекурсивно построить правое поддерево из части выражения справа от
   оператора.
3. Dернуть корень поддерева.
Пример: Рассмотрим выражение (2 + 3) * (4 - 1).
1. Находим оператор * с наименьшим приоритетом вне скобок. Он становится
   корнем.
2. Левое поддерево: (2 + 3)
   * Оператор + становится корнем.
   * Левый операнд: 2 (лист)
   * Правый операнд: 3 (лист)
3. Правое поддерево: (4 - 1)
   * Оператор - становится корнем.
   * Левый операнд: 4 (лист)
   * Правый операнд: 1 (лист)
5. Полученное дерево:
````C
*
/ \
+ -
/ \ / \
2 3 4 1
````
### Алгоритм вычисления значения по дереву

Вычисление значения выражения по дереву выполняется рекурсивно:
1. Базовый случай: Если узел – лист, вернуть значение операнда.
2. Рекурсивный шаг:
   * Рекурсивно вычислить значение левого поддерева.
   * Рекурсивно вычислить значение правого поддерева.
   * Выполнить операцию, соответствующую оператору в текущем узле, над
   результатами левого и правого поддеревьев.
3. Вернуть результат операции.
#### Пример (продолжение):
Корень *:
   * Левое поддерево +: 2 + 3 = 5
   * Правое поддерево -: 4 - 1 = 3
   * Результат: 5 * 3 = 15

### Реализация на C (упрощенная, для целых чисел и основных операций):
````C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    char op; // оператор или '0' для числа
    int value; // значение, если op == '0'
    struct Node *left, *right;
} Node;
    // ... (функции для создания узлов, парсинга и построения дерева)
int evaluate(Node* root) {
    if (root->op == '0') {
        return root->value;
}
    int leftValue = evaluate(root->left);
    int rightValue = evaluate(root->right);

    switch (root->op) {
        case '+': return leftValue + rightValue;
        case '-': return leftValue - rightValue;
        case '*': return leftValue * rightValue;
        case '/': return leftValue / rightValue;
        default: return 0; // обработка ошибок
    }
}
````

может требовать использования стека или других вспомогательных структур

Хеш-функция — функция, которая по ключу k выдаёт целое число h(k) в диапазоне [0, m−1], где m — размер таблицы.
Используется для быстрого определения «ячейки» таблицы, куда помещать элемент.

#### Свойства хорошей хеш-функции:
* Детерминированность: одинаковый ключ → одинаковый хеш
* Равномерность распределения: ключи равномерно распределяются по 0..m−1
* Быстрота вычисления
* Минимизация коллизий (но полностью избежать их невозможно)

Коллизия — ситуация, когда два разных ключа дают один и тот же хеш: h(k1) = h(k2), k1 ≠ k2.

Метод цепочек — самый распространённый способ разрешения коллизий.
В каждой ячейке таблицы хранится односвязный список (цепочка) элементов с одинаковым хешем.

#### Структура хеш-таблицы с цепочками на C
````C
#define TABLE_SIZE 10007   // простое число для лучшего распределения

typedef struct Entry {
    int key;               // ключ
    int value;             // значение (или void* для универсальности)
    struct Entry *next;
} Entry;

typedef struct {
    Entry *buckets[TABLE_SIZE];
} HashTable;
````

### Операции и их вычислительные сложности
(при хорошей хеш-функции и коэффициенте загрузки α ≈ 1)

* Поиск по ключу

Вычисляем хеш → идём по цепочке в бакете → сравниваем ключи.
Средняя сложность: O(1 + α) ≈ O(1)
Худшая (все в одной цепочке): O(n)

* Добавление элемента

Вычисляем хеш → вставляем в начало цепочки бакета (O(1)).
Средняя: O(1), худшая: O(n)

* Удаление элемента по ключу

Вычисляем хеш → ищем в цепочке → удаляем найденный узел.
Средняя: O(1), худшая: O(n)

### Реализация на C (min-heap, простая версия)
````C
unsigned int hash(int key) {
    unsigned int h = key;
    h = ((h >> 16) ^ h) * 0x45d9f3b;
    h = ((h >> 16) ^ h) * 0x45d9f3b;
    h = (h >> 16) ^ h;
    return h % TABLE_SIZE;
}

void ht_insert(HashTable *ht, int key, int value) {
    unsigned int index = hash(key);
    Entry *new = malloc(sizeof(Entry));
    new->key = key;
    new->value = value;
    new->next = ht->buckets[index];   // вставка в начало цепочки
    ht->buckets[index] = new;
}

int* ht_search(HashTable *ht, int key) {
    unsigned int index = hash(key);
    Entry *cur = ht->buckets[index];
    while (cur) {
        if (cur->key == key) return &cur->value;
        cur = cur->next;
    }
    return NULL;   // не найден
}

void ht_delete(HashTable *ht, int key) {
    unsigned int index = hash(key);
    Entry *cur = ht->buckets[index];
    Entry *prev = NULL;
    while (cur) {
        if (cur->key == key) {
            if (prev) prev->next = cur->next;
            else ht->buckets[index] = cur->next;
            free(cur);
            return;
        }
        prev = cur;
        cur = cur->next;
    }
}

Современные процессоры и компилятор C используют почти исключительно дополнительный код (two’s complement) для signed типов.
Однако на зачёте часто требуют знать все три исторических формата:

Прямой код — старший бит = знак (0 положительное, 1 отрицательное), остальные биты — модуль числа
Пример на 8 битах: +5 → 00000101, -5 → 10000101

Недостатки — два нуля (+0 и -0), сложение требует отдельной обработки знаков

Пример на 8 битах: +5 → 00000101, -5 → 11111010

Недостатки — всё ещё два нуля, при сложении часто нужна end-around carry коррекция

Дополнительный код (two’s complement) — положительные как в прямом коде, отрицательные = инверсия всех бит модуля + 1
Пример на 8 битах: +5 → 00000101, -5 → 11111011

Преимущества — один ноль, сложение и вычитание одинаковы для всех чисел, переполнение легко обнаруживается
Именно этот формат используется во всех современных CPU для signed char, short, int, long, long long
unsigned типы — всегда прямой двоичный код, арифметика по модулю 2ⁿ

Преобразование строкового представления числа в целочисленное значение является
распространенной задачей. 

Мы рассмотрим общий случай, когда число записано в
системе счисления с основанием base (например, для десятичной системы base = 10,
для двоичной base = 2, для шестнадцатеричной base = 16).

### Схема Горнера
Схема Горнера - это эффективный алгоритм для вычисления значения многочлена:
P(x) = a_n * x^n + a_{n-1} * x^(n-1) + ... + a_1 * x + a_0
В контексте преобразования строки в число, x - это основание системы счисления
(base), а коэффициенты a_i - это цифры числа, представленные в виде символов.
Например, число "123" в десятичной системе можно представить как многочлен 1 *
10^2 + 2 * 10^1 + 3 * 10^0.


### Основная идея схемы Горнера

Любое число в системе счисления с основанием p можно рассматривать как многочлен:

``dₙ₋₁ · pⁿ⁻¹ + dₙ₋₂ · pⁿ⁻² + … + d₁ · p + d₀``

Схема Горнера позволяет вычислить значение этого многочлена за один проход слева направо, используя только умножения и сложения:

``text((…((dₙ₋₁ · p + dₙ₋₂) · p + dₙ₋₃) · p + …) · p + d₀)``

Пример для "735" (p = 10):
→ ((7 × 10 + 3) × 10 + 5) = 735

Всего 2 умножения и 2 сложения вместо вычисления 10² и 10¹ отдельно.

Поэтому преобразование строки → число делается так:
``res = 0``
для каждого символа цифры d в строке слева направо:
``res = res * base + d``
Сложность: O(n) операций, где n — длина строки.

````
long long str_to_ll(const char *s, int base) {
    long long res = 0;
    int sign = 1;

    if (*s == '-') { 
        sign = -1; 
        s++; 
    } else if (*s == '+') s++;
    
    while (*s) {
        int digit;
        if (*s >= '0' && *s <= '9') digit = *s - '0';
        else if (*s >= 'A' && *s <= 'Z') digit = 10 + (*s - 'A');
        else if (*s >= 'a' && *s <= 'z') digit = 10 + (*s - 'a');
        else break; // неверный символ

        if (digit >= base) break; // цифра больше основания

        res = res * base + digit;   // ← это и есть схема Горнера
        s++;
    }
    return sign * res;
}
````


### Обратная схема Горнера
Нужна для преобразования числа → строка в заданной системе счисления.

Идея: последовательно выделяем младшие цифры через остаток от деления.
````
735 ÷ 10 = 73 остаток 5 → последняя цифра
73 ÷ 10 = 7 остаток 3 → следующая
7 ÷ 10 = 0 остаток 7 → первая цифра
Цифры получаются в обратном порядке → сохраняем во временный буфер → разворачиваем.
````
Сложность: O(logₚ |x|) = O(количество цифр в числе).


````
#include <string.h>

void ll_to_str(long long num, char *buf, int base) {
    if (num == 0) {
        strcpy(buf, "0");
        return;
    }

    int sign = 0;
    if (num < 0) {
        sign = 1;
        num = -num;
    }

    char temp[66];  // хватит для long long в двоичной системе
    int i = 0;

    while (num > 0) {
        int digit = num % base;
        if (digit < 10) temp[i++] = '0' + digit;
        else temp[i++] = 'A' + (digit - 10);
        num /= base;
    }

    if (sign) temp[i++] = '-';

    // разворот
    int j = 0;
    while (i > 0) buf[j++] = temp[--i];
    buf[j] = '\0';
}

### Выделение лексем из строки. Принцип границы и strtok
strtok — классическая, но ОПАСНАЯ функция (не потокобезопасна, модифицирует исходную строку).

#### Принцип работы:

Первый вызов strtok(str, delim) — ищет первый токен в str, заменяет разделитель на '\0', возвращает указатель

Последующие вызовы strtok(NULL, delim) — продолжают поиск с места остановки

Внутри использует статическую переменную → нельзя использовать одновременно на двух строках

Правильный цикл:
````
char s[] = "apple,orange;banana grape";
char *token = strtok(s, " ,;");
while (token != NULL) {
    printf("%s\n", token);
    token = strtok(NULL, " ,;");
}
````

Альтернатива без strtok:
````
char *my_strtok(char *str, const char *delim) {
    static char *pos = NULL;
    если str != NULL → pos = str;
    если pos == NULL → возврат NULL;

    // пропускаем разделители
    while (*pos && strchr(delim, *pos)) pos++;

    если (*pos == '\0') → возврат NULL;

    char *start = pos;
    while (*pos && !strchr(delim, *pos)) pos++;

    если (*pos) { *pos++ = '\0'; }  // ставим границу

    return start;
}
````

* static для локальной переменной (самое частое на зачёте)
Переменная сохраняет своё значение между вызовами функции.
Инициализируется один раз (при старте программы или при первом входе в блок).

Ограничивает область видимости текущим файлом перевода (translation unit).
Другие .c-файлы не видят её даже через extern.

Полезно для скрытия вспомогательных функций от остального проекта.
* static для члена структуры или объединения (с C11)
