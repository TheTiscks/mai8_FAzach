## Тип указателя. Арифметика указателей. Массивы и строки. Операции над строками и их непосредственная реализация

Указатель — это переменная, которая хранит адрес другой переменной в памяти. 

Указатели позволяют работать с данными напрямую через их адреса, что обеспечивает гибкость и эффективность при работе с памятью

### Синтаксис объявления:
``тип *имя;``
На каждой архитектуре все указатели имеют одинаковый размер.
#### Тип указателя влияет на интерпретацию данных и на шаг арифметики.

``& (взятие адреса): получение адреса переменной.``

``* (разыменование): доступ к значению переменной по ее адресу (получение значения, на которое указывает указатель).``

``Присваивание: указателю можно присвоить адрес другой переменной того же типа, либо значение NULL (пустой указатель, который ни на что не указывает).``

### Арифметика указателей
Разрешённые операции:

- прибавление/вычитание целого числа: p + n, p - n

- вычитание одного указателя от другого того же типа: p2 - p1 → расстояние в элементах

- сравнение указателей: ==, !=, <, >, <=, >= (имеет смысл только в пределах одного массива или объекта)

Ключевое правило:
p + n означает «адрес, который на n элементов типа *p дальше текущего».
Пример:
````
int a[10];
int *p = a;
p + 3;     // адрес a[3]
````
Если int занимает 4 байта, то реально адрес увеличивается на 12 байт (компилятор автоматически умножает n на sizeof(*p))

#### ! Запрещено складывать два указателя, умножать/делить указатели.

### Массивы и указатели
#### Массив – это набор элементов одного типа, расположенных в памяти последовательно. 
В C имя массива является указателем на его первый элемент.
```
int arr[5]; // Объявление массива из 5 целых чисел
arr[0] = 10; // Доступ к элементу массива по индексу
int *ptr = arr; // Указатель на первый элемент массива
````
##### В большинстве контекстов имя массива преобразуется в указатель на его первый элемент.
Исключения: оператор sizeof и унарный &.
````
int a[10];
a    ≡ &a[0]           // тип int*
a+1  ≡ &a[1]
*(a+5) ≡ a[5]
````
Передача массива в функцию всегда происходит по указателю:
``void f(int arr[])    // на самом деле: void f(int *arr)``

### Строки в C
Строка — это массив символов char, заканчивающийся нулевым байтом '\0' (нуль-терминатор).

Литерал строки автоматически получает '\0' в конце.
````
Cchar s1[] = "hello";     // 6 байт: h e l l o \0
char *s2 = "world";      // указатель на строковый литерал (только для чтения)
````


### Стандартные операции над строками (библиотека <string.h>) и их непосредственная реализация

``strlen`` — длина строки без '\0', Сложность O(n)

````
size_t my_strlen(const char *s) {
    size_t len = 0;
    while (s[len] != '\0') len++;
    return len;
}
````

#### strcpy — копирование строки (включая '\0')
````
char *my_strcpy(char *dest, const char *src) {
    char *start = dest;
    while ((*dest++ = *src++)) ;    // копируем до '\0' включительно
    return start;
}
````

#### strncpy — копирование не более n символов (опасна: не всегда ставит '\0')
Реализация вручную надёжнее:

````
char *my_strncpy(char *dest, const char *src, size_t n) {
char *start = dest;
while (n-- && (*dest++ = *src++)) ;
if (n == (size_t)-1) *dest = '\0';  // если дошли до лимита
return start;
}
````

#### strcat — конкатенация
````
char *my_strcat(char *dest, const char *src) {
char *start = dest;
while (*dest) dest++;        // ищем конец dest
while ((*dest++ = *src++)) ;
return start;
}
````

#### strcmp — лексикографическое сравнение
Возвращает <0, 0, >0
````
int my_strcmp(const char *s1, const char *s2) {
    while (*s1 && (*s1 == *s2)) {
        s1++; 
        s2++;
    }
    return *(unsigned char*)s1 - *(unsigned char*)s2;
}
````

### Важно! 

Строковые литералы находятся в области только для чтения — запись в них → UB

a[i] полностью эквивалентно *(a+i)

Указатель void* — универсальный, арифметика над ним запрещена

Разница между char s[100] и char *s — первая имеет фиксированный размер, вторая может указывать на динамическую память

Все функции из <string.h> требуют, чтобы строки были корректно нуль-терминированы
